Web 架构：

1、套用模版型，csdn / cnblog / github / 建站系统等
安全测试思路上的不同：一般以模版套用，基本模版无漏洞，大部分都采用测试用户管理权限为主

2、前后端分离
安全测试思路上的不同：前端以JS（Vue,NodeJS等）安全问题，主要以API接口测试，前端漏洞（如XSS）为主，后端隐蔽难度加大。

3、集成软件包宝塔，Phpstudy，xamp等
安全测试思路上的不同：主要是防护体系，权限差异为主，有些集成软件自身设置了高权限管理，有些没有
宝塔命令执行不行，文件管理除了web都能看，其他都不让看
绕过宝塔限制

4、自主环境镜像，云镜像打包，自行一个个搭建
安全测试思路上的不同：主要是防护体系，权限差异为主
命令执行可以，文件管理基本都能看 除去一些高权限目录（root目录等），或者可能人工设置
横向提权

5、容器拉取镜像，Docker
安全测试思路上的不同：虚拟化技术，在后期测试要进程逃逸提权
内置的虚拟化技术，单独的磁盘空间给应用做支撑
Docker 进程逃逸提权

6、纯静态页面，纯HTML+CSS+JS的设计
安全测试思路上的不同：无后期讲到的Web漏洞
搜搜相关找其他线索：找域名，资产，客户端等

正向代理

原理：正向代理帮客户端访问无法直接访问的资源，客户端知道代理的存在并主动配置
影响：目标服务器不知道真实的客户端是谁，只能看到代理服务器的 IP
主要用于突破访问限制、保护客户端隐私等
例如VPN

反向代理
原理：反向代理帮助服务器接收和处理客户端请求
影响：访问目标只是一个代理，非真实应用服务器，隐藏了真实服务器的信息
访问热门网站时，很可能在与反向代理打交道，而不是直接访问真实的服务器
客户端不知道反向代理的存在，以为自己在直接访问目标服务器
例如负载均衡等

注意：正向代理和反向代理都是解决访问不可达的问题，但由于反向代理中多出一个可以重定向解析的功能操作，导致反代理出的站点指向和真实应用毫无关系！

可以设置反向代理请求/bil到biliblil上，请求/bai到baidu上，就像中介一样

负载均衡

原理：分摊到多个操作单元上进行执行，共同完成工作任务
影响：有多个服务器加载服务，测试过程中存在多个目标情况
站长工具ping可以检测，只有几个IP的就是负载均衡

API接口

​​API 就是软件之间的“服务员”​​，帮你传话、跑腿、拿数据。API 的价值就是：让别人（比如顺丰）的功能，变成你的功能
能够获取目标网站一个查询的地方，

1、Web API:
通过HTTP协议进行通信的API，常用于Web服务和应用程序
例如，RESTful API、GraphQL API。

2、库和框架API:
提供特定编程语言或框架功能的API，供开发者在应用程序中使用
例如，Java API、Python标准库。

3、操作系统API:
提供操作系统功能访问的API
例如，Windows API、POSIX API

4、远程API:
允许在网络上远程访问服务的API
例如，SOAP API、XML-RPC API

内部API（公司内部用）​​
​​例子​​：
​​美团骑手​​调用​​美团APP订单系统​​的API（获取配送信息）
​​特点​​：不对外开放，只有美团自己的系统能用。

外部API（开放给所有人或合作伙伴）​​
​​例子​​：
​​美团APP​​ 调用​​微信支付API​​（微信付钱）
​​特点​​：任何人都可以申请使用（但可能需要付费或认证）。

有利影响：

利用（Exploitation）

密钥暴露的常见情况

  1. JavaScript 中的硬编码API
    密钥开发人员经常为了方便而将API密钥嵌入前端代码中，却忘记了任何人都可以使用浏览器开发人员工具查看此代码。

  2. 客户端框架中的加密密钥
    加密密钥有时会包含在前端库或脚本中，用于本地加密/解密数据。这些密钥很容易被提取并被恶意使用。

  3. 不安全的配置文件
    嵌入在 Web 应用程序中的配置文件可能包含纯文本的敏感凭据或密钥。

  4. 暴力破解密钥：
    弱密钥或可预测密钥（例如从时间戳或短字母数字字符串派生的密钥）可以通过使用 Hashcat 或John the Ripper等工具的暴力攻击来破解。

  5. 破解哈希：
    MD5和 SHA-1等过时的哈希函数容易受到碰撞和原像攻击等攻击。
    缺乏加盐使得攻击者能够使用彩虹表将哈希反转为纯文本。

  6. 客户端代码中暴露的键：
    前端代码中的硬编码加密密钥或API机密会将其暴露给任何有权访问该应用程序的人。
    这使得攻击者可以解密敏感数据或冒充用户。

  7. 位翻转攻击：
    未经身份验证的加密（例如，没有 MAC 的AES -CBC）使攻击者能够修改密文，从而对解密的明文进行可控的更改。
