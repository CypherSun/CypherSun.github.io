社会工程学

什么是社会工程学？

社会工程学：用来描述任何以人为目标的网络攻击的术语，会导致攻击者对目标的生活进行重大控制。社会工程学攻击是多层次的，并随着“滚雪球效应”逐步升级

其他形式的社会工程学

社会工程涵盖了任何依赖于诱骗人类向攻击者提供访问权限而不是直接攻击技术的攻击，与目标直接互动是社会工程学最常见的形式

社会工程学的极限取决于攻击者，优秀的社会工程师利用心理技巧，在任何可行的情况下使用大量心理技巧来攻击他们的目标

防范社会工程学攻击

要避免受到社会工程学攻击是非常困难的，因为攻击者并不总是直接接触，而是会找到能某个未经许就能给他们提供信息的人

设置多种身份验证方式
不要将不明来源的设备插入电脑
任何合法员工都不会要求提供密码或其他保护帐户的信息


社会工程学：网络钓鱼

网络钓鱼（Phishing）目标包括个人和企业。网络钓鱼是获取公司基础设施访问权限的初始攻击手段，之后进一步攻击公司网络，但现已有自动化工具来应对钓鱼威胁

什么是网络钓鱼？

网络钓鱼是社会工程学的一个分支。社会工程学用于描述任何利用人类而不是计算机系统的攻击，而网络钓鱼则专门描述攻击者向受害者发送短信、电子邮件或其他形式诱骗受害者打开恶意网页的攻击

网络钓鱼仅指电子邮件，该术语已演变为涵盖其他类别。这些其他形式被称为“短信钓鱼”（smishing）和“语音钓鱼”（vishing）。这些攻击非常普遍，会使用泄露或窃取的电话号码和电子邮件列表大规模部署

网络钓鱼通常运用心理欺骗手段，总是让受害者点击攻击者拥有的 Web 应用程序的链接，受害者会被要求输入敏感信息，此时恶意网站会存储这些信息。或者，受害者无意间从恶意页面安装恶意软件，从而为攻击者提供了进入其设备和网络的入口

网络钓鱼攻击主要有三种类型：

攻击类型 定义
常规网络钓鱼（General Phishing） 简单、大规模的网络钓鱼攻击，不针对任何特定目标，但可能针对大型群体。这些大规模攻击通常比较简单，比较容易识别，因为消息和恶意网站通常制作得不太好，并且包含许多可见的错误
鱼叉式网络钓鱼（Spearphishing） 比常规网络钓鱼更具针对性，其目标是个人或小群体。鱼叉式网络钓鱼活动比一般网络钓鱼中使用的通信和恶意网站更精心设计，它们旨在针对特定群体
捕鲸（Whaling） 鲸钓攻击比鱼叉式网络钓鱼更加具体，它针对的是高价值个人。鲸钓攻击的信息都经过精心设计，很难被发现

网络钓鱼攻击过程：

攻击者发送恶意钓鱼电子邮件
受害者收到邮件，其中一些人打开电子邮件并点击链接
受害者将其凭证输入攻击者的虚假网页
网页存储凭证或直接发送给攻击者
攻击者使用凭证访问该网站，从而接管受害者的账户

当恶意网页模仿现有知名网页时，钓鱼攻击的效果最佳。攻击者会使用工具来克隆现有页面，然后编辑该页面

识别钓鱼攻击

通用网络钓鱼攻击相对容易识别，它们的语法较差，不称呼受害者的姓名

恶意网站的域名与合法网站使用的域名相似，攻击者会注册看起来一样的域名，这是常见的策略

HTML 电子邮件也可用于掩盖正在使用的真实域名，将鼠标悬停在链接上来查看，真正的链接应显示在浏览器底部

基于电子邮件的网络钓鱼中的“发件人”电子邮件地址通常很可疑，通用的大规模网络钓鱼活动只会使用 Gmail 地址 ，而不会使用与合法网站使用的域名相似的恶意网站的域名

识别钓鱼邮件的最佳方式是保持警惕，寻找任何可疑的东西。除了最专业的钓鱼邮件，大多数都会有某种错误

防范钓鱼攻击

删除未知或不受信任的电子邮件
不要打开不受信任的电子邮件的附件
不要点击电子邮件或消息中的嵌入链接
确保设备和防病毒软件是最新
避免公开个人信息

任何人都有可能受到钓鱼攻击

恶意软件和勒索软件

恶意软件（malware）定义为任何旨在代表攻击者执行恶意操作的软件

攻击者会利用恶意软件窃取信息、造成破坏或在受感染的系统上执行任意命令。恶意软件常用于执行“命令和控制”（Command and Control，C2 或 C&C）任务。C2 恶意软件会重新连接到等待的服务器，使攻击者能够远程控制受感染系统

勒索软件

勒索软件是特殊的恶意软件，用于尽可能多的感染系统，加密设备上的数据并索要赎金

勒索软件通过利用软件中的已知漏洞进行传播，一旦感染开始，其传播速度极快，并可能索要数百万赎金。勒索软件攻击的目标是感染尽可能多的系统。使用只有攻击者知道的密钥加密尽可能多的数据，使这些数据无法访问。在支付赎金后，恶意软件可能会或不会解密数据并自毁，这取决于攻击者的手段。

传播方式

攻击者会将文件作为编译后的.exe、PDF、.ps1 PowerShell 脚本或.bat 批处理脚本、HTML 应用程序文件 (.hta) 甚至 .js JavaScript 文件发送，以便由 Windows 内置的 JavaScript 解释器执行

攻击者通过多种方式和格式向受害者发送代码，一旦代码执行，感染便开始

攻击者会利用公司环境中面向公众的基础设施中的漏洞，进入内部网络，并在恶意软件的帮助下发起更大规模的攻击

保持安全

接受操作系统等重要软件提供的更新和补丁
不要点击电子邮件中的可疑链接
警惕那些通过电子邮件或消息试图下载或运行文件的人
不要将未知的媒体设备插入重要计算机
始终备份重要数据
确保杀毒软件始终保持最新且激活状态

密码和身份验证

什么是强密码？

强密码包含大小写字符、符号和数字，长度超过八个字符，并且其信息仅限于密码拥有者掌握。由于密码与个人信息相关，通过社会工程或对目标的深入信息收集仍可能被破解

目前的最佳实践更强调密码的长度而非复杂性，通过使用短语而不是传统密码，密码长度显著增加，同时仍保留一些复杂性。这种方法更容易记忆，同时仍极难通过暴力破解。

理想情况下，应使用长且完全随机的密码，是客观上最安全的选择

什么是弱密码？

短密码在暴力破解攻击面前非常脆弱

与密码强度同等重要的是避免密码重复使用。如果在多个账户上重复使用强密码且该密码被泄露，攻击者只需在所有受影响的帐户上使用相同的强密码即可

已泄露的密码

当注册一个账户，服务提供商必须存储密码副本，以便下次登录时进行验证。

将密码存储为明文不可行，因为如果数据库被黑客入侵，每个密码都会泄露。
加密密码是一个改进，但改进不大。如果密码是加密存储的，那么它们也可以被解密。攻击者只需获取密钥，就可以将每个加密密码还原成明文。

行业标准的密码存储方法称为哈希密码。哈希密码是指使用复杂的数学算法，将任何输入转化为唯一的，固定长度的输出，而且无法逆转。当注册时，密码会被哈希处理并存储在数据库中，任何人都无法读取。
当尝试登录时，系统会对输入的密码应用相同的算法：如果存储的哈希值与登录的密码的哈希值匹配，则被视为成功通过身份验证

理想情况下，每个服务都应使用安全的算法对用户密码进行哈希处理。即使数据库泄露，攻击者仍需大量时间和计算资源去尝试暴力破解哈希找到明文密码。密码越长，包含的字符种类越多，要猜中生成哈希密码所需的计算量就越大。在这种情况下，密码是安全的，但电子邮件地址或用户名仍可能会被泄露

最坏的情况下，攻击者不仅知道用户名，还知道密码，从而可以接管账户或进行“凭证填充”攻击，将用户名和密码组合用于其他服务。这些包含凭证的泄露数据库常出现在暗网上

密码攻击

攻击者在攻击密码和身份验证系统时有几种选择。有些攻击完全是本地攻击，其他攻击则是涉及原始服务器的远程攻击。

本地攻击需要窃取相关凭证的副本。攻击者将获取包含窃取的用户名/电子邮件和哈希密码的文件，然后使用软件尝试猜测生成该哈希值并输入，使用随机生成的字符序列和预先生成的密码字典，攻击者课将现有的字典进行变形处理，添加新字符、符号或随机元素

远程攻击分为两类；一类是尝试通过向服务器发送请求并查看其响应来暴力破解已知用户名，一类是使用从先前泄露事件中获得的已知用户名和密码对来查看它们在目标网站上是否有效

多因素认证和密码管理器

通过使用密码管理器和多因素认证来进一步提升密码安全性

多因素认证

多因素认证（Multi-Factor Authentication，MFA）用于描述任何需要多个因素才能登录的身份验证过程的术语

第二个认证要素被称为基于时间的一次性密码（Time-based One Time Password，TOTP）

基于 SMS 的 TOTP 认证是最常用的，却不是最安全，因为攻击者可重新路由受害者的短信

基于时间的一次性密码仍然是大多数用户最方便的第二认证因素，大多数服务允许使用“认证器应用”（Authenticator App）来进行 MFA。系统会提供二维码和“密钥”，可扫描二维码或将密钥导入 Authy 或 Google Authenticator 等应用程序。将密钥添加到应用中，这些应用可以直接在设备上生成 TOTP 代码，无需任何网络连接或可被拦截的短信

密码管理器和生成强密码

密码管理器提供了一个安全的密码的存储空间。它们将密码存储在“保险库”中。加密存储空间，可以保存在设备上或作为在线服务。这些保险库通过主密码访问

每个密码管理器都有其独特的优缺点，常见的密码管理器包括：

1Password
LastPass
KeePass
Bitwarden


公共网络安全

公共 WiFi 为攻击者提供了机会去攻击其他用户的设备或拦截和记录网络流量以窃取敏感信息的机会。攻击者利用大多数人希望看到公共网络可用的心理，攻击者可快速建立网络并监控每个连接的流量，这被称为“中间人”攻击，且非常容易实施

现网站一般使用传输层安全协议（TLS）来加密服务器与用户之间的通信，使这种攻击的发生几率显著降低，但连接到任何网络都会使自己设备对网络上的其他人可见

解决方案

不要连接到不受信任的网络，使用移动热点或专用网络总是更安全

虚拟专用网络（VPN）会加密离开和重新进入的所有流量，使任何拦截手段变得无效，因为被拦截的数据会变成乱码。免费 VPN 往往无法提供最佳安全性，并且经常收集数据以牟利

网站连接的安全性

所有网站都应在加密连接的保护下传输信息。与使用 VPN 一样，这可以防止攻击者在拦截网络流量时读取或修改网络流。用于创建 超文本传输​​协议（HTTPS）的加密连接称为传输层安全性（TLS），在大多数浏览器中，搜索栏左侧的挂锁表示该连接是安全的

挂锁图标表明连接是安全的，但它不能保证网站本身是安全的。恶意网站仍然可以轻松获得 TLS 证书，但这并不能阻止该网站具有恶意目的。

如果访问的网站没有挂锁图标或使用不可信网络时，不要输入任何凭据或敏感信息

挂锁有一个十字或感叹​​号，这表示连接理论上是安全的，但服务器使用的证书存在问题。此类图标可能意味着证书过期，或攻击者在干扰连接安全

在尝试访问网页时，也可能遇到页面安全提示，这意味着加密连接存在问题，可能导致流量被窃取

备份

备份是简单的保险措施，可在最坏的情况下可挽回损失

黄金 3-2-1 备份规则

备份的黄金标准是“3-2-1 规则”

3-2-1 规则
应始终保留至少三份最新的数据副本，其中可以包括原始副本，但必须保留所有副本。
备份应存储在至少两种不同的存储介质上
一个（或多个）备份应“异地”存储

备份的频率同样重要，备份的频率自行决定，取决于数据的敏感度、泄露风险和可用的备份空间大小

更新和补丁

软件更新

更新允许开发人员添加新功能、修复漏洞或调整产品。在软件中发现安全漏洞时，开发人员会发布特殊更新，称为“补丁”，其中包含针对漏洞的修复程序或以其他方式来“修补”安全问题

及时更新操作系统等关键软件，这些系统中的漏洞可能特别危险

所有软件最终都会失去维护者的支持，进入“寿命终止”状态（EOL），变得弃用并且不再接收更新。

杀毒软件更新

大多数杀毒软件都会频繁更新，因为它们主要使用已知漏洞签名的本地数据库来工作，而该数据库必须保持最新

当发现新的恶意软件时，它会被发送给杀毒软件厂商，厂商会生成一个“签名”来识别这一特定恶意软件的。这些签名会分发给全球使用杀毒软件的每台设备，确保安装的杀毒软件能及时发现所有最新恶意软件

如果不允许防病毒软件更新，它仍然可以通过其他方法捕获一些恶意软件。但是，本地签名数据库很快就会过时，导致恶意软件可能从漏洞中漏网。
