社会工程学

什么是社会工程学？

社会工程学：以人为目标的网络攻击的统称，社会工程学攻击是多层次的，并随着“滚雪球效应”逐步升级

其他形式的社会工程学

社会工程学涵盖了依赖于欺骗人的攻击，而不是直接攻击技术的攻击，与目标直接互动是社会工程学最常见的形式

社会工程学的极限取决于攻击者，优秀的社会工程师利用心理技巧，在任何可能的情况下“黑掉”目标

防范社会工程学攻击

要完全防范社会工程学攻击是非常困难的，因为攻击者并不总是直接接触，而是可能会找到某个未经许可就能提供他们信息的人

始终确保设置多种身份验证方式，并确保服务提供商严格遵守这些验证
切勿将外部介质连接到电脑，而是将其交给当地警方妥善处理
通过已知的电话号码或电子邮件地址确认收到的电话或消息是否真实
正规员工永远不会询问密码或其他保护账户的信息


社会工程学：钓鱼攻击

钓鱼攻击（Phishing）目标包括个人和企业。钓鱼攻击是进入公司基础设施的初始攻击手段，之后会进一步攻击公司网络，现已有自动化工具来应对钓鱼威胁

什么是钓鱼攻击？

钓鱼攻击是社会工程学的一种形式。社会工程学是一个广泛的术语，用于描述任何利用人类而非计算机系统的攻击，而钓鱼则专指诈骗者或其他攻击者通过发送短信、电子邮件或其他在线信息诱骗受害者打开恶意网页的攻击方式。

“钓鱼”主要指电子邮件攻击，这个术语已扩展涵盖其他类型的通讯。这些形式被称为“短信钓鱼”（smishing）和“语音钓鱼”（vishing）。这些攻击非常普遍，会使用泄露或窃取的电话号码和电子邮件地址进行大规模散播

钓鱼消息通常运用心理操控技巧，并包含一个指向由攻击者控制的网页链接。受害者点击链接后，会被要求输入敏感信息，此时恶意网站会保存这些信息，攻击便完成。另一种情况是，受害者会无意间从恶意网页上安装恶意软件，从而为攻击者打开进入设备和网络的入口

钓鱼攻击主要三种类型：

攻击类型 定义
普通钓鱼（General Phishing） 一种简单的、广泛散布的钓鱼攻击，没有特定目标，通常针对大型用户群。这些大规模活动通常比较简单，且通常比较容易识别，因为消息和恶意网站通常不够精细，且常常含有明显的错误。
针对性钓鱼（Spearphishing） 比普通钓鱼更具针对性，目标是某个特定个人或小群体。由于针对特定群体，且常作为更大攻击活动的一部分，针对性钓鱼的消息和恶意网站通常更精心设计，比普通钓鱼更难识别。
捕鲸（Whaling） 比针对性钓鱼更具体，目标是高价值个人。这类信息通常非常精细、难以识别

攻击过程：

攻击者发起恶意钓鱼邮件活动
潜在受害者收到邮件——部分人打开邮件并点击链接
受害者在攻击者伪造的网页上输入自己的凭证
该网页会保存这些凭证，或直接将其发送给攻击者
攻击者使用这些凭证访问网站，从而接管受害者的账户
钓鱼攻击在恶意网页仿照现有网页时效果最佳，攻击者通常会使用免费工具直接克隆现有页面，以随意编辑

钓鱼攻击的最终目标可能因攻击者的身份而异

识别钓鱼攻击

普通钓鱼攻击相对容易识别，它们的语法通常较差，且通常不使用受害者的姓名。某些钓鱼邮件可能极难识别，有些甚至足以欺骗专业人士

钓鱼邮件的情境会显得非常可信，在针对性钓鱼或捕鲸攻击中，情境会精心设计贴合目标的特点

恶意网站的域名会与合法网站的域名非常相似。攻击者会注册看起来几乎一样的域名，这是非常常见的手法

HTML 邮件也可用来掩盖实际使用的域名，将鼠标悬停在链接上查看实际链接，链接会在屏幕底部显示

邮件钓鱼的“发件人”地址通常很可疑，许多大规模钓鱼活动会使用 Gmail 地址，而不是使用被模仿公司关联的域名

识别钓鱼邮件的最佳方式是保持警惕，留意任何可疑之处。除了最专业的钓鱼邮件，大多数都会有某种错误

防范钓鱼攻击

删除不明或不可信的邮件，如果邮件内容有任何可疑之处，可将其报告为垃圾邮件，或者在工作场合将其转发给 IT 安全部门
切勿打开来自不可信邮件的附件，包括任何意外收到的附件，即使来自可信联系人
不要点击邮件或消息中的嵌入链接
确保设备和防病毒软件保持更新
避免公开个人信息
任何人都有可能受到钓鱼攻击

恶意软件和勒索软件

恶意软件（malware）是指任何为攻击者执行恶意操作的软件。恶意软件的种类繁多，恶意软件和勒索软件。

一旦安装，攻击者通常利用恶意软件窃取信息、造成破坏或在受感染系统上执行任意命令。恶意软件通常用于执行“指挥与控制”（Command and Control，简称 C2 或 C&C）任务。C2 恶意软件会连接到一个等待中的服务器，使攻击者能够远程控制受感染系统，通常包括键盘记录等功能作为恶意软件的内置部分。

勒索软件

勒索软件是特殊的恶意软件，旨在尽可能多地感染系统，锁定设备上的数据并以此勒索赎金

勒索软件通常通过利用常用软件中的已知漏洞传播；一旦感染开始，传播速度极快，赎金金额可能高达数百万美元。勒索软件攻击的目标是感染尽可能多的系统，通过加密数据使之不可访问，而加密密钥仅掌握在攻击者手中。在支付赎金后，恶意软件可能会解密数据并自毁，但是否解密完全取决于攻击者。

传播方式

攻击者可能会将文件以 .exe、PDF、.ps1 PowerShell 脚本、.bat 批处理脚本、HTML 应用程序文件（.hta）.js JavaScript 文件的形式发送，使 Windows 的 JavaScript 解释器执行代码

攻击者可通过多种方式和格式向受害者发送代码，一旦代码执行，感染便开始

攻击者可能会利用企业环境中公共基础设施的漏洞，为自己打开通道进入内网，进而通过恶意软件实施更大规模的攻击

保持安全

防范恶意软件最好的方法是结合安全意识和保持系统更新

始终接受更新和补丁，尤其是操作系统等重要软件。更新通常包含安全漏洞的修复
切勿点击可疑链接，尤其是在电子邮件中。尽量避免打开文件附件
警惕任何试图下载或运行文件的行为，尤其是通过电子邮件发送的文件
不要将不明来源的设备插入电脑
始终备份重要数据，这对从勒索软件攻击中恢复至关重要
确保杀毒软件始终保持最新和激活状态


密码和身份验证

什么是强密码？

密码包含大小写字符、符号和数字，长度超过八个字符，并且其信息仅限于密码拥有者掌握。它也没有遵循常见的模式，因此更难猜测。由于密码与个人信息相关，通过社会工程或对目标的深入信息收集仍可能被破解

目前的最佳实践更强调密码的长度而非复杂性

通过使用短语而不是传统密码，密码长度显著增加，同时仍保留一些复杂性，尽管看起来没有那么难辨认。这种方法更容易记忆，同时仍然极难通过暴力破解

理想情况下，应使用长且完全随机的密码

这种密码需要数百万年才能破解，是客观上最安全的选择。其缺点在于使用不便

什么是弱密码？

人们会选择对自己有意义的简单密码，常常遵循“简单”

较短的密码在暴力破解攻击面前也非常脆弱

与密码强度同等重要的是避免密码重复使用。即使有一个非常强的密码，但如果在多个账户上重复使用且该密码被泄露，攻击者可以轻松利用相同的密码攻击所有受影响的账户。如果发现密码被泄露，必须更改所有账户的密码

已泄露的密码

并不是每个服务都能安全地存储密码。

当注册一个账户，服务提供商必须存储密码副本，以便下次登录时进行验证。以明文形式存储密码并不可行，因为一旦数据库被黑客入侵，每个密码都会立即泄露。加密密码是一个改进，但提升不大。如果密码是加密存储的，那么它们也可以被解密——攻击者只需要获取密钥，就可以将每个加密密码还原成明文。

业界标准的密码存储方法称为哈希密码。哈希密码是指使用复杂的数学算法，将任何输入转化为一个唯一的固定长度输出，而这一过程不可逆。这意味着当注册时，密码会被哈希并存储在数据库中，使任何人都无法读取。 当尝试登录时，系统会对输入的密码应用相同的算法：如果存储的哈希值与试图登录的密码的哈希值匹配，则视为成功通过认证。 理想情况下，每个服务都应使用安全的算法对用户密码进行哈希处理。即使整个数据库泄露，攻击者仍需浪费大量时间和计算资源去尝试暴力破解哈希，以找到明文密码。密码越长，包含的字符种类越多，要猜中生成哈希值的密码输入所需的计算量就越大。在这种情况下，密码是安全的，但电子邮件地址或用户名仍可能会被泄露

在最坏的情况下，明文密码可能立即暴露，或者攻击者很容易找到。如果发生这种情况，攻击者不仅知道用户名，还知道密码，从而可以接管账户，或进行“凭证填充”攻击——将用户名和密码组合用于其他服务，看看是否在其他地方复用了密码。这些包含凭证的泄露数据库经常出现在暗网上

最大的、最知名的数据泄露查询工具称为“Have I Been Pwned?”。它是一个免费的在线服务，搜集并整理了各种数据泄露的信息，用户可以在其中输入自己的电子邮件地址，查看是否出现在任何泄露事件中。

密码攻击

攻击者在攻击密码和认证系统时有几种选择。一些攻击完全是本地的，另一些则是涉及原始服务器的远程攻击

本地攻击需要窃取的凭证副本。攻击者希密会获取包含用户名/电子邮件和哈码的文件，然后使用软件尝试猜测生成该哈希值的输入，方法是使用随机生成的字符序列或使用预先生成的密码字典。
攻击者将现有的字典进行变形处理，添加新字符、符号或随机元素

远程攻击通常分为两类；一种是尝试对已知用户名进行暴力破解，通过向服务器发送请求并观察其响应，另一种是使用从先前泄露事件中获得的已知用户名和密码组合，测试它们在目标网站上是否有效

多因素认证和密码管理器

通过使用密码管理器和多因素认证来进一步提升密码安全性

多因素认证

多因素认证（Multi-Factor Authentication，MFA）是指需要多个验证要素的认证过程。第二个认证要素被称为“基于时间的一次性密码”（Time-based One Time Password，TOTP）

实现多因素认证时，会考虑以下三种主要认证因素：

拥有的东西
知道的东西
生物特征

理想情况下，认证系统会使用所有三种因素。

大多数个人用户无法获得智能卡读卡器、昂贵的硬件认证器或指纹扫描仪。不过，大多数都可以使用智能手机接收或生成基于时间的一次性密码。基于手机的 MFA 成为个人用户最常用的多因素认证方式

基于 SMS 的 TOTP 认证虽使用最广泛，却不是最安全，因为攻击者可重新路由受害者的短信

基于时间的一次性密码仍然是大多数用户最方便的第二认证因素

大多数服务允许使用“认证器应用”（Authenticator App）来进行 MFA。系统会提供一个二维码和一个“密钥”，可扫描二维码或将密钥导入 Authy 或 Google Authenticator（适用于 Android 和 IOS）等应用程序。一旦将密钥添加到应用中，这些应用可以直接在设备上生成 TOTP 代码，无需任何网络连接或可被拦截的短信

密码管理器和生成强密码

密码管理器在最基本的层面上提供了一个安全存储密码的空间。它们将密码存储在“保险库”中：一种加密存储空间，可以保存在设备上，也可以作为在线服务。这些保险库通过主密码访问，这是唯一需要记住的密码或通过生物特征数据

功能更全面的密码管理器可快速生成非常强的密码并自动保存，然后，当登录应用时，密码管理器可以自动输入密码，全程无需离开应用程序

密码管理器是管理众多账户的推荐工具，整个系统的安全性可能依赖于单一的主密码，所以确保主密码足够强大！

常见的密码管理器包括：

1Password
LastPass
KeePass
Bitwarden

每个密码管理器都有其独特的优缺点


公共网络安全

问题

依赖公共 WiFi 实际上可能非常危险

公共 WiFi 虽非常方便，但也为攻击者提供了机会去攻击其他用户的设备或拦截并记录网络流量以窃取敏感信息。攻击手法相对简单，利用大多数人认为公共网络随处可用的心理，攻击者可以快速架设一个自己的网络，监视所有连接到该网络的设备的流量；这种攻击被称为“中间人攻击”，且非常容易实施。

现代网站一般使用传输层安全协议（TLS）来加密服务器与用户之间的通信，使这种攻击的发生几率显著降低。连接到任何网络都会让设备暴露在其他网络用户面前

解决方案

尽量不要连接到不可信的网络，使用移动热点或私人网络始终是更安全的选择

虚拟专用网络（VPN）可以加密从设备发出的所有流量，使任何拦截手段变得无效，因为被拦截的数据会变成毫无意义的乱码。免费的 VPN 通常无法提供最好的安全性，往往会收集数据以盈利

网站连接的安全性

所有网站现在都应该仅在加密连接的保护下传输信息。与使用 VPN 类似，这能防止攻击者在拦截流量时读取或修改网络通信。创建安全超文本传输协议（HTTPS）所使用的加密连接称为传输层安全协议（TLS），这种安全连接会在地址栏左侧显示一个小锁图标，表示连接已加密

有了 TLS 的保护，流量只能在极少数情况下被解密

锁图标表明连接是加密的，但不保证网站本身是安全的。一个恶意网站仍然可以拥有 TLS 证书，但这并不阻止该网站的恶意目的。

如果访问的网站没有锁图标，不要输入任何凭证或敏感信息，尤其使用不可信网络时

看到锁图标上有一个斜杠或感叹号，这表示连接理论上是加密的，但服务器证书存在某些问题。此类图标可能意味着服务器管理员让证书过期了，或者可能是攻击者在干扰连接安全。如果图标不是常规的锁形，不要信任连接的安全性

在尝试访问网页时，也可能遇到完整页面错误提示，如果看到类似错误，应该点击“返回”按钮，这通常意味着加密连接存在问题，可能导致流量被窃取

备份

备份是简单的保险措施，在最坏的情况下可以挽回多倍的损失

黄金 3-2-1 备份规则

备份的黄金标准是“3-2-1 规则”

3-2-1 规则的内容如下：

应始终保留至少三份最新的数据副本；其中可包括原始数据，但所有副本必须保持更新。

备份应存储在至少两种不同的存储介质上；例如：一个云端备份和一个 USB 设备，也可以包括电脑硬盘。

至少有一个备份应存储在“异地”。例如，Google Drive 等云服务对于个人用户来说是理想的异地备份选择。

满足了 3-2-1 规则的条件，备份就应该是安全的，备份的频率同样重要

备份的频率自行决定，通常取决于数据的敏感性、被破坏的风险以及备份空间的可用性


更新和补丁

软件更新

更新是软件开发生命周期的重要部分，允许开发人员添加新功能、修复漏洞或调整产品的某些功能。当在软件中发现安全漏洞时，开发人员会发布特别更新，称为“补丁”，用来修复或“填补”安全漏洞

应尽可能及时更新软件，尤其是操作系统等关键软件，因为这些系统中的漏洞可能特别危险

所有软件最终都会失去维护支持，进入“寿命终止”状态（EOL），不再接收更新。应尽快替换该软件。如果无法替换，则应尽可能隔离该设备，以防止日后发现但无法修复的漏洞被利用

杀毒软件更新

大多杀毒软件会频繁地更新；因为它们主要依赖本地已知漏洞签名数据库，该数据库必须保持最新

当发现新的恶意软件时，它会被发送给杀毒软件厂商，厂商生成一个识别该恶意软件的“签名”。这些签名随后被分发给全球使用该杀毒软件的所有设备，确保杀毒软件始终能识别最新的恶意软件

如果杀毒软件未能更新，仍可通过其他方法捕捉到部分恶意软件。然而，本地签名数据库会迅速过时，可能导致某些恶意软件被漏检
