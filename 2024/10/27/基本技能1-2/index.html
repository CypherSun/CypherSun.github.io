SSH 访问 Linux

什么是 SSH 及其工作原理？

Secure Shell（SSH）是一种设备间的加密协议。通过使用加密技术，以人类可读的格式发送的任何输入会在网络传输过程中加密 —— 当到达远程机器时再进行解密，是连接和操作远程 Linux 机器命令行的常用方式

SSH 允许远程在另一台设备上执行命令，任何在设备间发送的数据在通过网络时都会被加密

使用 SSH 需要两个信息：
远程机器的 IP 地址
远程机器上有效账户的登录凭证

输入 yes 信任主机，输入账户的密码。连接成功后，任何执行的命令将会在远程机器上运行

在 SSH 登录中输入密码时，不会看到任何文本或符号不会有任何反馈，只需输入密码后回车登录

输入exit断开连接

标志和开关简介（flags and switches）

大多数命令都允许使用参数，这些参数由连字符和一个特定的关键字标识，这些关键字被称为标志或开关。在使用命令时，除非另有指定，否则它将执行默认行为。

使用 ls -a 命令查看所有文件和文件夹，包括以 . 开头的隐藏文件和文件夹
使用 ls –help 命令列出 ls 命令列出提供简短的描述和使用示例的使用参数选项

使用 man 命令

手册页（man page）包含 Linux 系统命令和应用程序的信息，可直接在 Linux 上或网站访问

使用 man 命令来查看手册页，可搭配其他命令一起使用

man 命令搭配其他命令可以查看 Linux 上几乎所有的命令，使用上下箭头阅读，q 退出

文件系统交互命令

命令 描述
touch 创建新文件
mkdir 创建新文件夹/目录（make directory）
cp 复制文件或文件夹（copy）
mv 移动文件或文件夹或重命名（move）
rm 删除文件或文件夹（remove）
file 确定文件的类型

以上所有命令都可搭配使用完整的文件路径

使用 touch 和 mkdir 命令

使用 touch 命令创建创建新文件
使用 mkdir 命令创建新文件夹/目录

使用 echo 命令或 nano 等文本编辑器在文件中添加内容

使用 rm 命令

使用 rm 命令删除文件
使用 rm -R 命令删除文件夹

使用 cp 和 mv 命令

使用 cp 命令将文件内容复制到新的文件中，第一个参数为已存在文件的名称，第二个参数为复制新文件指定的名称。如果目标文件不存在，将自动创建新文件
使用 cp -r 命令将文件夹及其内容复制到新的文件夹中，第一个参数为已存在文件夹的名称，第二个参数为复制新文件夹指定的名称。如果目标文件夹不存在，将自动创建新文件夹

使用 mv 命令将文件或文件夹移动到新文件夹，或者对第二个参数提供的文件或文件夹名称进行重命名，不会创建新文件

使用 file 命令

使用 file 命令确定文件或文件夹的类型，返回文件的类型描述


权限基础知识

使用 ls -lh 命令列出目录中所有文件的详细信息，包括文件权限、大小、所有者、修改时间等

文件或文件夹有几种特性，特性决定了允许执行哪些操作，以及哪个用户或用户组有权限执行操作：

读取 (Read)
写入 (Write)
执行 (Execute)

用户和用户组之间的区别

Linux 的优点在于权限非常细化，通过设置权限，一组用户可以对同一个文件拥有相同或不同的权限，而不会影响文件的所有者

使用 su 命令

通过目标用户的用户名和密码进行切换账户

使用 su 命令需要输入用户的密码来切换到指定的用户账户，默认情况下不会加载该用户的完整环境变量
使用 su -l 命令需要输入用户的密码，它会完全切换到指定用户的实际登录环境，加载所有新用户的属性等

如果没有指定用户，默认情况下会切换到 root 用户。

常见目录

/etc

/etc 目录（“etc” 代表 etcetera）是存放操作系统使用的配置和系统文件位置

在 /etc 文件夹中包含 passwd 和 shadow 文件，/etc/passwd 文件存储了用户账户的信息，/etc/shadow 文件存储了每个用户的加密哈希密码。系统常使 SHA-512 或其他哈希算法对密码进行加密

/var

/var 目录（“var” 代表 variable data）是存放由系统上运行的服务或应用程序访问、修改或生成的数据

/root

/root 目录是 root 用户系统管理员的主目录，它与普通用户的主目录 /home 不同，/root 是专门为 root 用户保留的

/tmp

/tmp 目录（“tmp” 代表 temporary）是用于存放仅需临时访问的文件和数据。一旦系统重启，/tmp 目录中的内容会被清除。/tmp 目录在默认情况下任何用户都有写入权限，/tmp 是存放枚举脚本和其他工具的好位置
