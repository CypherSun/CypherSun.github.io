侦察

侦察：是指发现并收集关于系统和受害者的信息。侦察阶段是对手计划攻击的阶段。

开源情报（OSINT）也属于侦察范畴。OSINT 是攻击者在进行后续攻击步骤之前必须完成的第一步。攻击者需要通过收集关于公司及其员工的所有可用信息，来研究目标，以便确定最佳的攻击目标。

为了在这个阶段进行操作，攻击者需要进行 OSINT

电子邮件收集：是通过公共、付费或免费服务获取电子邮件地址的过程。攻击者可以利用电子邮件收集进行网络钓鱼攻击。在侦察阶段，攻击者有大量的工具可用，其中一些包括：

theHarvester：除了收集电子邮件，这个工具还能通过多个公共数据源收集姓名、子域、IP 和 URL
Hunter.io：这是一个电子邮件搜寻工具，可以获取与特定域名相关的联系信息
OSINT Framework：提供了基于各种类别的 OSINT 工具集合

攻击者还会利用社交媒体网站，来收集想要攻击的特定受害者或公司的信息。这些社交媒体上的信息对于攻击者进行网络钓鱼攻击可能非常有帮助。

武器化

在顺利完成侦察阶段后，大多数攻击者通常会使用自动化工具生成恶意软件，或者从暗网上购买恶意软件。这种工具会将恶意软件和漏洞利用结合成一个可传递的有效载荷。而那些更复杂的攻击者或由国家资助的高级持续性威胁（APT）组织则会编写自定义的恶意软件，以确保其独特性，从而避开目标上的检测。

恶意软件：一种用于破坏、干扰或未经授权访问计算机的程序或软件。
漏洞利用：利用应用程序或系统中的漏洞或缺陷的程序或代码。
有效载荷：攻击者在系统上执行的恶意代码。

在武器化阶段，攻击者可能会进行以下操作：
创建一个包含恶意宏或 VBA（Visual Basic for Applications）脚本的感染 Microsoft Office 文档
攻击者可以创建恶意有效载荷或非常复杂的蠕虫病毒，将其植入 USB 驱动器，然后在公共场所分发
攻击者可能会选择使用指挥与控制（C2）技术，在受害者的计算机上执行命令或传递更多有效载荷
攻击者会选择后门植入方式

传递

在传递阶段，有几种常见选择：

钓鱼邮件：在进行侦察并确定目标后，恶意攻击者会精心编写一封恶意邮件，目标可能是特定个人（鱼叉式钓鱼攻击）或公司内部的多名人员。邮件内包含有效载荷或恶意软件

在公共场所投放受感染的 USB 驱动器：例如在咖啡店、停车场或街道上。攻击者可能会通过印有公司 logo 的 USB 驱动器实施高级 USB 投放攻击，并假装是客户，将这些设备作为礼物寄送给公司

水坑攻击：这是一种专门针对特定群体的攻击方式，攻击者会入侵目标群体经常访问的网站，并将其重定向到恶意网站。攻击者会寻找并利用网站的已知漏洞。为了提高攻击成功率，攻击者可能会发送看似“无害”的电子邮件，引导受害者访问目标网站并点击恶意链接。受害者访问网站后，会无意中下载恶意软件或恶意应用程序，这种攻击类型被称为“驱动式下载”

利用

为了获得系统的访问权限，攻击者需要利用系统中的漏洞。在获得系统访问权限后，恶意攻击者可能会利用软件、系统或服务器漏洞来提升权限，或者在网络中进行横向移动。横向移动是指恶意攻击者在获得初始访问权限后，使用各种技术进一步深入网络，以便获取敏感数据。

在这个阶段，攻击者也可能使用“零日漏洞利用”。零日漏洞利用或零日漏洞是一个在尚未被发现的漏洞，它暴露了软件或硬件中的安全漏洞，可能在问题显现之前造成严重影响。零日漏洞在一开始几乎没有检测机会。

以下是攻击者执行漏洞利用的常见方式：
受害者通过打开邮件附件或点击恶意链接来触发漏洞
使用零日漏洞进行攻击
利用软件、硬件，甚至是人性的弱点
攻击者利用服务器漏洞进行攻击

安装

后门可以让攻击者绕过安全机制并隐藏访问权限。后门也被称为“访问点”。

一旦攻击者成功获取了系统访问权限，攻击者会希望能够在连接丢失、被检测到或初始访问权限被移除，或者系统被修补后，仍能重新访问系统。此时，攻击者需要安装一个持久性的后门。持久性后门可以确保攻击者能够重新访问之前已被攻陷的系统。

以下是实现持久性的几种方式：
在 Web 服务器上安装 Web Shell：Web Shell 是一种由攻击者编写的恶意脚本，使用 ASP、PHP 或 JSP 等 Web 开发语言，目的是维持对被攻陷系统的持续访问。由于 Web Shell 的简单性和文件格式，它们可能难以被检测并被误认为是无害文件。

在受害者的机器上安装后门：使攻击者可以远程控制受害者的机器并执行恶意代码。

创建或修改 Windows 服务：攻击者可以创建或修改 Windows 服务，使其能够定期执行恶意脚本或有效载荷，作为保持持久性的一部分。攻击者可以使用 sc.exe和 Reg 这些工具来修改服务配置。攻击者还可能伪装恶意有效载荷，使用与操作系统或合法软件相关的服务名称。

在注册表的“运行键”或启动文件夹中为恶意有效载荷添加条目：通过这种方式，每次用户登录计算机时，有效载荷都会被执行。启动文件夹既有针对单个用户账户的位置，也有系统范围的启动文件夹，无论哪个用户登录，都会被检查。

此外，攻击者还可能在这一阶段使用时间篡改技术（Timestomping），以避免被取证调查人员检测，并让恶意软件看起来像合法程序的一部分。时间篡改技术允许攻击者修改文件的时间戳，包括修改、访问、创建和更改的时间。

指挥与控制

在成功建立持久性并在受害者计算机上执行恶意软件后，攻击者通过该恶意软件打开了 C2（指挥与控制）通道，以便远程控制和操纵受害者。这一通信类型也被称为 C&C 或 C2 信标，是指 C&C 服务器与被感染主机之间的一种恶意通信方式。被感染的主机会持续与 C2 服务器通信，这就是“信标”一词的由来。

被攻陷的终端将与攻击者设置的外部服务器通信，以建立指挥与控制通道。一旦连接建立，攻击者即可完全控制受害者的计算机

攻击者通常使用以下 C2 通道：
HTTP（端口 80）和 HTTPS（端口 443）协议：这种信标方式将恶意流量与合法流量混合，帮助攻击者规避防火墙的检测

DNS（域名服务器）：被感染的机器会不断向归属于攻击者的 DNS 服务器发送 DNS 请求，这种 C2 通信方式也被称为 DNS 隧道

对手或者其他被攻陷的主机也可能是 C2 基础设施的拥有者。

达成目标

在经历了攻击的前六个阶段后，这意味着攻击者将对最初设定的目的采取行动。通过直接访问系统，攻击者可以执行以下操作：

收集用户的登录凭证
执行权限提升
进行内部侦察
在公司网络环境中横向移动
收集并外泄敏感数据
删除备份和影子副本，影子副本是微软的一项技术，可以创建文件或卷的备份副本和快照
覆盖或损坏数据
