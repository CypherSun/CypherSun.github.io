使用 SSH 访问 Linux 机器

什么是 SSH 及其工作原理？

Secure Shell（SSH）是一种设备间的加密协议。通过使用加密技术，以人类可读的格式发送的任何输入会在网络传输过程中加密 —— 当到达远程机器时再进行解密，是连接和操作远程 Linux 机器命令行的常用方式。

SSH 允许远程在另一台设备上执行命令，任何在设备间发送的数据在通过网络（如互联网）时都会被加密。

使用 SSH 的语法只需两个信息：
远程机器的 IP 地址
远程机器上有效账户的正确凭证

使用的命令是 ssh，然后是账户用户名，接着是 @ 和机器的 IP 地址

信任主机，输入账户的密码。连接成功后，任何执行的命令将会在远程机器上运行，而不是在本地机器上

在 SSH 登录提示符中输入密码时，不会有任何可见的反馈。在输入密码时不会看到任何文本或符号，只需输入密码后按回车登录。

标志和开关简介

大多数命令都允许使用参数。这些参数通常由连字符和一个特定的关键字标识，这些关键字被称为标志（flags）或开关（switches）。在使用命令时，除非另有指定，否则它将执行默认行为。

使用 “ls -a” 命令查看所有文件和文件夹，包括以 “.” 开头的隐藏文件和文件夹
使用 “ls –help” 命令列出 ls 命令可接受的所有选项，并提供简短的描述和使用示例

手册页

手册页（man page）包含了 Linux 命令和应用程序的详细文档，是关于系统命令和 Linux 上应用程序的信息，可以直接在机器上或者在线访问

使用 “man” 命令来查看手册页，列出可以与 “ls” 一起使用的选项

man搭配命令可以查看Linux 上几乎所有的命令，在使用上下箭头键滚动阅读手册页，按 q 退出

文件系统交互继续

命令 描述
touch 创建一个新的空文件
mkdir 创建一个新的文件夹/目录（make directory）
cp 复制文件或文件夹（copy）
mv 移动文件或文件夹或重命名（move）
rm 删除文件或文件夹（remove）
file 确定文件的类型

与使用 cat 命令类似，在以上所有这些命令中都可使用完整的文件路径

创建文件和文件夹（touch, mkdir）

使用 “touch” 命令创建一个新的空文件
使用 “mkdir” 命令创建一个新的文件夹/目录

想要在该文件中添加内容，使用 echo 命令或 nano 等文本编辑器

删除文件和文件夹 (rm)

使用 “rm” 命令删除一个文件
使用 “rm -R” 命令删除一个文件夹

复制和移动文件和文件夹 (cp, mv)

使用 “cp” 命令将现有文件的所有内容复制到新的文件中，第一个参数为已存在文件的名称，第二个参数为复制时新文件指定的名称。如果目标文件不存在，将自动创建新文件。
使用 “cp -r” 命令将现有文件夹及其所有内容复制到新的文件夹中，第一个参数为已存在文件夹的名称，第二个参数为复制时新文件夹指定的名称。如果目标文件夹不存在，将自动创建新文件夹。

使用 “mv” 命令将文件或文件夹移动到新文件夹，或者对第二个参数提供的文件或文件夹进行重命名，不会创建新文件

确定文件类型

使用 “file” 命令确定文件或文件夹的类型，会返回文件的类型描述


权限基础知识

使用 “ls -lh” 命令列出目录中所有文件的详细信息，包括文件权限、大小、所有者、修改时间等

对于确定一个文件或文件夹的某些特性，以及是否有访问权限。文件或文件夹可以有几种特性，这些特性决定了允许执行的操作，以及哪个用户或用户组有权限执行这些操作：
读取 (Read)
写入 (Write)
执行 (Execute)

用户和用户组之间的区别

Linux 的一个优点在于权限可以非常细化，尽管某个用户技术上是文件的所有者，但通过设置权限，一组用户可以对同一个文件拥有相同或不同的权限，而不会影响文件的所有者。

需要以下切换用户账户：
目标用户的用户名
目标用户的密码

使用 “su” 命令需要输入用户的密码来切换到指定的用户账户，默认情况下不会加载该用户的完整环境变量。
使用 “su -l” 命令需要输入用户的密码，它会完全切换到指定用户的实际登录环境，加载所有新用户的属性，比如环境变量等。

如果没有指定用户，默认情况下会切换到 root 用户。

常见目录

/etc

/etc 目录是系统中最重要的目录之一。/etc 文件夹（“etc” 代表 etcetera）是存放操作系统使用的配置文件和系统文件的常见位置。

在 /etc 文件夹中，通常包含 passwd 和 shadow 文件。/etc/passwd 文件存储了用户账户的信息，而 /etc/shadow 文件存储了每个用户的密码的加密哈希。系统通常使用 SHA-512 或其他安全哈希算法来对密码进行加密

/var

/var 目录是 “variable data”（可变数据）的缩写，是 Linux 中的重要目录。这个目录用于存储由系统上运行的服务或应用程序频繁访问、修改或生成的数据

/root

/root 目录是root用户系统管理员的主目录，它与普通用户的主目录 /home 不同。/root 是专门为 root 用户保留的，因此被放置在根目录下，而不是 /home 目录中

/tmp

/tmp 目录是 Linux 系统中的一个特殊目录，用于存储临时文件。/tmp 是 “temporary”（临时）的缩写，用于存放仅需临时访问的数据。一旦系统重启，/tmp 目录中的内容会被清除

对于渗透测试人员而言，/tmp 目录的一个优势在于，默认情况下任何用户都有写入权限。这意味着在获得对系统的访问权限后，/tmp 是一个存放枚举脚本和其他工具的理想位置。
