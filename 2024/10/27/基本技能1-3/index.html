终端文本编辑器

Nano

nano 是一个易于使用的文本编辑器，输入 nano 和文件名来创建或编辑文件

使用 “向上” 和 “向下” 箭头键在各行之间导航，或者使用 “Enter” 键开始新行

Nano 具备一些易于记忆的功能，涵盖了对文本编辑器的常用需求，包括：

搜索文本
复制和粘贴
跳转到指定行号
查看当前所在的行号

可以通过按 “Ctrl” 键（显示为 ^）和对应的字母来使用这些功能

Ctrl + W：搜索文本
Ctrl + K：剪切行
Ctrl + U：粘贴行
Ctrl + _：跳转到指定行号
Ctrl + C：查看当前行号
Ctrl + X：退出 nano，在退出前会提示是否保存更改

VIM

VIM 文本编辑器，显著优势包括：

可定制性：可以根据自己的需求修改快捷键和配置
语法高亮：VIM 支持语法高亮显示，对于编写或维护代码非常有帮助，是许多软件开发者的首选
广泛兼容性：VIM 可以在几乎所有的终端上运行，而某些系统可能默认没有安装 nano
丰富的学习资源：VIM 拥有大量的学习资料，包括速查表、教程和社区支持，方便新手逐步掌握

常用/实用工具

文件下载（Wget）

使用 “wget” 命令通过 HTTP 协议从网页上下载文件，只需提供目标资源的完整 URL 地址。

从主机传输文件 - SCP（SSH）

安全复制（SCP）是一种通过 SSH 协议在两台计算机之间传输文件的安全方法。不同于普通的 cp 命令，scp 提供了身份验证和加密。

scp 使用“来源”和“目标”的模式，需要当前系统和远程系统的用户名和密码
scp 可以将文件或目录从当前系统复制到远程系统，或者从远程系统复制到当前系统。

使用 “scp” 命令从本地系统复制文件到远程系统

使用 “scp” 命令从远程系统复制文件到本地系统

从主机提供文件 - WEB

Python 提供了一个轻量且易于使用的模块，称为 http.server，它可以将计算机快速变成一个简易的 Web 服务器，用于提供文件。其他计算机可以通过诸如 curl 和 wget 之类的命令来下载这些文件。

使用 Python 3 的 http.server 模块会在当前目录下提供文件，并且可以通过一些可选参数来更改目录

Python 3 服务器默认在端口 8000 上运行，因此在使用 wget 命令下载文件时，需要指定该端口

需要打开一个新终端来使用 wget，而运行 Python3 Web 服务器的终端必须保持打开状态
因为 Python3 Web 服务器启动后会在该终端中持续运行，直到手动取消

http.server 模块没有目录索引功能，因此必须确切知道要下载的文件的名称和路径。

虽然 http.server 足够简单易用，但它没有索引功能。还有一些功能更强大的轻量级 Web 服务器，比如 Updog

进程基础知识

进程是计算机上运行的程序，由内核进行管理。每个进程都有一个唯一的进程标识符（PID），PID 会按照进程启动的顺序递增。

查看进程

使用 “ps” 命令显示当前用户会话中运行的进程列表，以及一些附加信息
使用 “ps aux” 命令查看所有用户的进程以及不属于当前会话的进程

使用 “top” 命令实时显示系统上运行的进程信息，而不是一次性的快照。这些统计数据会自动刷新，默认每 10 秒刷新一次。

通过 “top” 命令可以更详细地了解系统的实时运行状态。

管理进程

使用 “kill” 命令提供要终止进程的 PID，通过发送信号来终止进程

在终止进程时，可以使用不同的信号类型来指定终止方式：

SIGTERM (15)：请求进程终止，但允许它在终止前执行清理任务。默认情况下，kill 命令会发送这个信号
SIGKILL (9)：直接强制终止进程，不允许其进行任何清理操作。适用于无法正常停止的进程
SIGSTOP (19)：停止或挂起进程，但不终止它。可以稍后使用 SIGCONT 恢复执行

进程是如何启动的？

操作系统（OS）利用命名空间将计算机的可用资源分配给各个进程。位于某一块中的进程能够使用该块中的一部分计算资源，这只是系统所有进程所能使用资源的一部分。

命名空间的一个主要优点是安全性，它可以隔离进程——只有位于同一命名空间中的进程才能彼此可见。

在系统启动时，ID 为 0 的进程是最早创建的进程，这个进程通常是系统的 init 进程。在 Linux 中，init 通常是 systemd，它负责启动和管理系统的所有其他进程，是所有用户进程的父进程。

进程/服务在系统启动时自动运行

有些应用程序可以在系统启动时自动运行，这些程序通常很重要，管理员会配置它们在系统启动时自动启动。

使用 “systemctl” 命令与 systemd 守护进程交互来管理这些服务

通过systemctl，可以执行四种操作：

start（启动）
stop（停止）
enable（启用）
disable（禁用）

Linux中的后台与前台运行

在 Linux 中，进程可以运行在前台或后台

使用 “&” 操作符执行命令并在后台运行
使用 “fg” 命令可以将挂起或在后台的进程恢复到前台
使用 “bg” 命令可以将挂起的进程继续在后台运行

使用 Ctrl + Z 会将前台进程挂起，但不会终止它


系统维护：自动化

Crontab 是一个具有特定格式的特殊文件，cron 进程会识别它，并逐行执行其中的每一条命令。使用 crontab 来管理和执行定时任务。cron 是一个在系统启动时运行的进程，用于管理和执行 crontab 中定义的任务。

Crontab 需要设置 6 个特定的字段：

值 描述
MIN 在每小时的第几分钟执行（0-59）
HOUR 在每天的几点钟执行（0-23）
DOM 在每月的哪一天执行（1-31）
MON 在每年的哪个月份执行（1-12）
DOW 在每周的哪一天执行（0-7，0和7表示星期日）
CMD 要执行的实际命令

通过 crontab -e 命令打开文本编辑器来编辑定时任务
使用 * 通配符来表示任意值，不指定特定的时间字段


系统维护：软件包管理

当开发者希望将软件提交给社区时，会将其提交到一个“apt”软件库中。程序和工具将被发布给广大用户。在这里，Linux 的两个最显著的优势得以展现：用户可访问性和开源工具的优点。

在 Linux 机器上使用 ls 命令时，这些文件充当了网关或注册表的角色。

虽然操作系统供应商通常会维护自己的软件库，但您也可以将社区软件库添加到您的列表中！这使您可以扩展操作系统的功能。可以通过 add-apt-repository 命令或列出另一个提供商来添加额外的软件库

管理您的软件库（添加和移除）

添加软件库使用 add-apt-repository 命令，通过添加新的软件库，可以访问更多的软件包。
安装软件使用 apt 命令，apt 是一个软件包管理工具，包含了一整套工具用于管理软件包和软件源，同时可以安装或移除软件包。apt 的优势在于每次更新系统时，会自动检查并更新已添加的软件库中的软件包。

软件库：存放软件包的集合，包含数百或数千个软件包
软件包：可安装的软件单元，包含特定程序的所有文件和依赖项

add-apt-repository：用于添加或移除软件库的工具，这会将新的软件库地址添加到系统的 sources.list.d 目录中，并自动更新 GPG 密钥
apt：用于管理软件包的安装和更新的工具

系统维护：日志

日志文件存放在 /var/log 目录中。这个目录包含了系统上运行的应用程序和服务生成的日志信息。操作系统能自动管理这些日志，这一过程称为 “日志轮替”（log rotation）。日志轮替会自动归档和清理旧的日志文件，以防止日志文件占用过多的磁盘空间。

在 Ubuntu 系统上运行的三项服务的日志：
Apache2 Web 服务器的日志：包括访问日志（access.log）和错误日志（error.log），记录了每一个 Web 请求的信息。
Fail2ban 服务的日志：用于监控和防止暴力破解行为，日志文件通常是 fail2ban.log。
UFW（Uncomplicated Firewall）防火墙的日志：UFW 是一个常用的防火墙工具，日志文件通常位于 ufw.log，记录了所有的防火墙规则和活动。

这些服务日志是监控系统健康状况和安全性的有效工具。特别是像 Web 服务器这样的服务日志，能够记录每一个请求的信息，这让开发人员和系统管理员能够诊断性能问题或调查入侵行为。

除了应用程序和服务的日志，系统还会生成一些日志文件，用于记录操作系统的运行状态和用户行为。
