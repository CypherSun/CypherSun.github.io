侦察

侦察是攻击者在攻击前的计划阶段，目的是发现并收集关于目标系统和受害者的信息。它包括开放源情报（OSINT），这是攻击者实施后续攻击的第一步。通过收集公司和员工的所有公开信息，攻击者可以确定最佳的攻击目标。

在侦察阶段，攻击者通常尚未明确具体的攻击目标，因此需要借助 OSINT 手段来获取信息。

电子邮件收集是侦察的重要组成部分，涉及从公开、付费或免费服务中获取电子邮件地址。攻击者利用收集的电子邮件发起网络钓鱼攻击。在侦察阶段，攻击者可以使用多种工具，包括：

theHarvester：收集电子邮件、姓名、子域名、IP地址和URL，利用多个公共数据源。
Hunter.io：电子邮件搜索工具，用于获取与特定域名关联的联系信息。
OSINT Framework：提供多种OSINT工具，分类方便使用。

攻击者还会利用社交媒体网站收集受害者或目标公司的信息

武器化

在侦察阶段结束后，攻击者进入武器化阶段，将恶意软件和漏洞利用结合为一个可投递的负载。大多数攻击者使用自动化工具生成恶意软件，或通过暗网获取现成的恶意工具。高级持续性威胁（APT）或国家支持的攻击者则可编写定制的恶意软件，使其更难被检测。

恶意软件（Malware）：旨在破坏、干扰或未经授权访问计算机的程序。
漏洞利用（Exploit）：利用应用或系统漏洞的代码。
负载（Payload）：攻击者在系统中执行的恶意代码。

在武器化阶段，攻击者会采取以下行动：

恶意文档：创建带有恶意宏或VBA脚本的受感染Microsoft Office文档。
物理传播：将带有恶意负载的U盘散布在公共场所，以传播蠕虫病毒。
命令与控制（C2）：配置C2技术，便于在受害者设备上执行指令或分发更多负载。
后门植入：部署后门程序，以便持续访问受害系统。


投递

投递阶段是攻击者选择传送恶意负载或恶意软件方式的阶段，旨在将恶意软件引入目标系统。

网络钓鱼邮件：在侦察并确定目标后，攻击者设计带有恶意负载的邮件，发送给特定个人或公司内部的多个员工。

分发带有恶意软件的 U 盘：攻击者可能在咖啡店、停车场等公共场所放置带有恶意软件的U盘，或通过更复杂的方式，比如印上公司标志并伪装成客户礼物，将U盘邮寄给目标公司。

水坑攻击：攻击者入侵目标群体常访问的网站，利用网站漏洞将其重定向至恶意网址。此类攻击称为“驱动式下载”，当受害者访问受感染网站时，恶意软件会被无意间下载至受害者设备上。

漏洞利用

为了获得系统访问权限，攻击者会利用系统中的漏洞。在获取初始访问权限后，攻击者可能利用软件、系统或服务器的漏洞进行权限提升或横向移动。横向移动是指攻击者在网络内进一步扩展，以获取敏感数据的技术。

攻击者可能在此阶段使用零日漏洞利用。零日漏洞指的是在现实环境中尚未被发现的漏洞，这些漏洞暴露了软件或硬件中的安全缺陷，往往在未被意识到之前已被恶意利用，且最初几乎无法检测。

漏洞利用的常见方式：

受害者通过打开电子邮件附件或点击恶意链接触发漏洞
使用零日漏洞利用进行攻击
利用软件、硬件，人类的弱点
触发服务器端的漏洞利用

安装

后门允许攻击者绕过安全措施并隐秘地访问系统。当攻击者获得系统访问权限后，为确保在连接中断、初始访问被清除或系统打补丁后仍可重新访问，攻击者会安装一个持久性后门。

持久性实现方式：

安装 Web Shell：在 Web 服务器上安装 Web Shell，使攻击者能持续访问。由于 Web Shell 的简单性及文件格式，常被误认为正常文件而难以检测。

安装后门程序：直接在受害者机器上安装后门程序。

创建或修改 Windows 服务：攻击者可通过创建或修改 Windows 服务来定期执行恶意脚本或负载，使用工具修改服务配置，甚至伪装成合法的服务名称。

添加启动条目：在注册表的 “Run 键” 或启动文件夹中添加恶意负载，使其在用户每次登录时自动执行。启动文件夹有用户账户级和系统级，系统级的条目会在所有用户登录时执行。

攻击者还可能使用 Timestomping 技术规避取证检测，通过修改文件的时间戳，使恶意软件伪装成合法程序的一部分。

指挥与控制

在获得持久性并成功在受害者机器上执行恶意软件后，攻击者通过 C2（指挥与控制）通道远程控制受害设备。这种持续通信方式也称为 C&C 或 C2 信标通信，指 C&C 服务器与受感染主机上的恶意软件之间的定期通信。受感染主机会与C2服务器持续联系，这也是“信标”一词的来源。

受感染设备会连接到攻击者的外部服务器，开启指挥与控制通道。一旦建立连接，攻击者便可以完全控制受害者的机器

常见的现代 C2 通道：

HTTP（端口80）和 HTTPS（端口443）：通过混合恶意流量和正常流量，帮助攻击者绕过防火墙检测。
域名系统（DNS）：受感染主机会不断向攻击者的 DNS 服务器发送 DNS 请求，这种 C2 方式称为 DNS 隧道。

C2 基础设施的拥有者可以是攻击者本身，也可以是另一个受感染的主机。

达成目标（数据外泄）

在经过前六个攻击阶段后，攻击者最终进入达成目标阶段，实现最初的攻击意图。通过完全控制受害者系统，攻击者可以执行以下操作：

收集用户凭证信息
提升权限
进行内部侦察
横向移动至其他系统
收集并外泄敏感数据
删除备份和卷影副本
覆盖或破坏数据
