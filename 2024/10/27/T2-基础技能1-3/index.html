终端文本编辑器

使用 Nano 文本编辑器

nano 是简单文本编辑器，输入 nano 和文件名来创建或编辑文件

使用上下键，回车开始新行

Nano 可用功能包括：
搜索文本
复制和粘贴
跳转到指定行号
查看当前所在的行号

按 Ctrl（显示为 ^）和字母使用功能：
Ctrl + W：搜索文本
Ctrl + K：剪切行
Ctrl + U：粘贴行
Ctrl + _：跳转到指定行号
Ctrl + C：查看当前行号
Ctrl + X：退出 nano

VIM

VIM 是高级文本编辑器，显著优势包括：

可定制：可根据自己需求修改快捷键和配置
语法高亮：支持语法高亮显示，对编写或维护代码非常有帮助，是首选
兼容：可在几乎所有终端运行，某些系统可能默认没安装 nano
学习资源：拥有大量的学习资料，包括速查表、教程和社区支持等

常用/实用工具

使用 Wget 命令

使用 wget 命令通过提供的目标资源 URL 地址， 使用 HTTP 协议从网页上下载文件

从主机传输文件 - SCP（SSH）

安全复制（SCP）是一种通过 SSH 协议在两台计算机之间传输文件的方法，SCP 提供了身份验证和加密

使用 scp “来源”和“目标”的模式，需要当前系统和远程系统的用户名和密码
scp 可以将文件或目录从当前系统复制到远程系统，或者从远程系统复制到当前系统。

使用 scp 命令和地址从本地系统复制文件到远程系统

使用 scp 命令和地址从远程系统复制文件到本地系统

从主机提供文件 - WEB

Python 提供了 http.server 模块，它可以将计算机变成一个简易 Web 服务器用于提供文件，其他计算机可以通过命令来下载文件

使用 Python3 的 http.server 模块会在当前目录下提供文件，并且可通过参数更改目录
运行 Python3 Web 服务器的终端必须保持打开状态，因为 Python3 Web 服务器启动后会在终端持续运行，直到手动取消

Python3 Web 服务器默认在端口 8000 上运行，另一台在使用命令下载文件时，需要指定该端口

http.server 模块没有目录索引功能，因此必须确切知道要下载的文件的名称和路径

进程基础知识

进程是计算机上运行的程序，由内核进行管理。每个进程都有唯一的进程标识符（PID），PID 会按照进程启动的顺序递增

查看进程

使用 ps 命令提供正在运行的进程列表和附加信息
使用 ps aux 命令查看其他用户运行的进程以及未从会话运行的进程

使用 top 命令实时显示系统上运行的进程信息，不是一次性的快照。这些统计数据自动刷新，默认每 10 秒刷新一次，可以更详细地了解系统的实时运行状态

管理进程

使用 kill 命令提供要终止进程的 PID，通过发送信号来终止进程

使用不同的信号类型来指定终止方式：

SIGTERM (15)：进程终止，但允许它在终止前进行清理操作。默认 kill 命令会发送这个信号
SIGKILL (9)：强制终止进程，不允许其进行清理操作，适用于无法正常停止的进程
SIGSTOP (19)：停止或挂起进程但不终止，可使用 SIGCONT 恢复执行

进程是如何启动的？

操作系统（os）使用命名空间将计算机上可用的资源最终分配给进程。命名空间的优点是安全性，它可隔离进程，只有位于同一命名空间中的进程才可见

在系统启动时，ID 为 0 的进程是最早创建的进程，这个进程通常是系统的 init 进程。
init 进程通常是 systemd，负责启动和管理系统的所有其他进程，是所有用户进程的父进程

进程/服务在系统启动时自动运行

管理员会配置有些应用程序在系统启动时自动运行

使用 systemctl 命令与 systemd 进程进行交互和管理

systemctl可以执行四种操作：

start（启动）
stop（停止）
enable（启用）
disable（禁用）

Linux中的后台与前台运行

进程可以以两种状态运行：后台和前台

使用 & 操作符执行命令并在后台运行
使用 ctrl + z 将前台进程挂起
使用 fg 命令可以将挂起或在后台的进程恢复到前台
使用 bg 命令可以将已挂起的进程恢复为在后台继续运行


系统维护：自动化

cron：
cron 在系统启动时运行，用于管理和执行定时任务
cron 会定期检查是否有预定任务需要执行，如果有，它就会执行任务
cron 负责执行所有已定义的任务，这些任务的定义来源是 crontab 文件

crontab：
crontab 是包含定时任务定义的特殊文件，任务按照特定的时间格式设定，告诉 cron 在什么时间运行哪些任务
crontab 格式可被 cron 进程识别，并可逐步执行每行

crontab 需要 6 个特定值：

值 描述
MIN 在每小时的第几分钟执行（0-59）
HOUR 在每天的几点钟执行（0-23）
DOM 在每月的哪一天执行（1-31）
MON 在每年的哪个月份执行（1-12）
DOW 在每周的哪一天执行（0-7，0和7表示星期日）
CMD 要执行的实际命令

通过 crontab -e 命令打开文本编辑器来编辑定时任务
使用 * 通配符来表示任意值，不指定特定的时间字段


系统维护：软件包管理

Linux 最显著的优势：用户可访问性和开源工具的优点。当开发者可以将软件提交给社区，会将其提交到一个“apt”软件库中，程序和工具将被发布给广大用户。在 Linux 上使用 ls 命令时，这些文件充当了网关或注册表的角色。

使用 add-apt-repository 和 apt 命令

软件库：存放软件包的集合，包含很多软件包
软件包：可安装的软件单元，包含特定程序的所有文件和依赖项

add-apt-repository：用于添加或移除软件库的工具，这会将新的软件库地址添加到系统的 sources.list.d 目录中，并自动更新 GPG 密钥
apt：用于管理软件包的安装和更新的工具，包含整套工具用于管理软件包和软件源，优势在于每次更新系统时，会自动检查并更新已添加的软件库中的软件包

使用 add-apt-repository 命令添加第三方提供的软件库，包含官方库中没有的软件包或最新版本
使用 apt 命令安装和卸载软件包，更新已安装的软件包

添加第三方软件，需要手动添加 GPG 密钥和软件库地址，以确保下载内容的完整性。GPG（GNU Privacy Guard）密钥是开发者提供的安全验证工具，用于确保下载的软件包是来自可信任的来源。如果密钥与系统信任的密钥和开发者使用的密钥不匹配，软件将无法下载。

系统维护：日志

日志文件存放在 /var/log 目录中，包含了系统上运行的应用程序和服务生成的日志信息。操作系统能自动管理这些日志，称为 “日志轮替”（log rotation）。日志轮替会自动归档和清理旧的日志文件，以防止日志文件占用过多的磁盘空间。

服务日志能监控系统健康状况和安全性，系统还会生成日志文件，记录操作系统的运行状态和用户行为
