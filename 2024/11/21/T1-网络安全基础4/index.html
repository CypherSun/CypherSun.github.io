侦察（Reconnaissance）

从攻击者的角度看，侦察是网络攻击的起点，指收集目标系统或个人信息以制定攻击计划的过程。

开源情报（OSINT）是侦察阶段的关键部分。攻击者通过公开渠道收集信息，例如公司的规模、电子邮件地址、电话号码，以及员工信息等，从而确定攻击目标和方法。

在准备攻击链的最后阶段前，必须首先完成侦察。在此过程中，需要通过 OSINT 收集关键信息。

电子邮件收集是常见的一步。攻击者通过公共或付费平台获取目标的电子邮件地址，并利用这些信息发起网络钓鱼攻击，通过伪造信息窃取登录凭据等敏感数据。攻击者在侦察阶段可能会使用以下工具：

theHarvester：收集电子邮件、子域、IP、URL 等信息
Hunter.io：搜索与特定域名相关的联系信息
OSINT Framework：提供分类良好的 OSINT 工具集合

攻击者还会利用 LinkedIn、Facebook、Twitter 等社交媒体平台挖掘目标的公开信息

武器化（Weaponization）

完成侦察后，攻击者进入武器化阶段，设计攻击载荷（payload）。武器化是将恶意软件与漏洞利用代码结合成可投递的载荷的过程。为减少直接互动的风险，攻击者通常依赖自动化工具生成恶意软件或通过暗网购买现成工具。而高级攻击者或 APT 组织则倾向于开发定制化恶意软件，以规避检测。

恶意软件（Malware）：破坏、干扰或未经授权访问系统的程序或软件
漏洞利用（Exploit）：利用系统或应用程序漏洞的代码
载荷（Payload）：在目标系统上执行的恶意代码

攻击者可能采取的行动：
创建嵌入恶意宏或 VBA 脚本的 Microsoft Office 文档，利用文档作为攻击媒介
制作复杂的恶意载荷或蠕虫病毒，将其植入 USB 驱动器，然后公开分发
使用指挥与控制（C2）技术远程执行命令或发送更多恶意载荷，持续控制目标设备
部署后门程序（backdoor），绕过安全机制，获得对目标系统的持续访问权限

投递（Delivery）

在投递阶段，攻击者决定如何将恶意载荷传递给目标。常用的方法包括：

网络钓鱼邮件：攻击者设计带有恶意载荷的邮件，针对个人，定向钓鱼攻击或组织内多名员工。例如：攻击者发现 A 公司销售部门的 Nancy 经常点赞 B 公司经理 Scott 的 LinkedIn 帖子，推测两人可能通过工作邮件交流。攻击者可以伪造了一封以 Scott 名义发送的“发票”邮件，伪装域名，使其看似来自 Scott 公司，实则包含恶意载荷。

分发感染的 USB 设备：攻击者将感染恶意程序的 USB 设备故意丢弃在公共场所，如咖啡店或停车场或邮寄给目标公司。更复杂的攻击手法是印上目标公司的标志，将设备伪装为“礼物”邮寄给公司

水坑攻击：攻击者入侵目标群体常访问的网站，利用漏洞将其重定向至恶意网站。攻击者还可能发送含恶意链接的邮件，引导受害者访问被劫持的网站。一旦受害者访问，恶意软件可能通过伪造的弹窗或扩展程序被下载，进行“驱动下载攻击”（Drive-By Download）。

漏洞利用（Exploitation）

为了获得系统访问权限，攻击者会利用目标的漏洞。在这一阶段，攻击者能展展示创造性：

攻击者发送包含伪造 Office 365 登录页面链接的钓鱼邮件，诱导受害者输入凭据。发送带有恶意宏附件的邮件，附件触发勒索软件的执行。受害者如果点击恶意链接并打开恶意文件，即可攻击成功。

攻击者进入系统后，可能利用软件、系统或服务器中的漏洞提升权限，并在网络中横向移动以获取更深层的访问权限。横向移动是攻击者深入网络以窃取敏感数据的关键技术。

攻击者可能使用“零日漏洞”，即尚未被发现和修补的漏洞。零日漏洞利用因其隐蔽性，在攻击初期几乎无法被检测，可能导致严重后果。

攻击者如何进行攻击的示例：
受害者打开恶意邮件附件或点击恶意链接
使用零日漏洞实施攻击
利用软件、硬件或人性的弱点
触发服务器漏洞进行利用

安装（Installation）

在安装阶段，攻击者需要确保持久性访问，即使连接丢失、访问被检测或系统被修补后仍能重新进入目标系统。这通常通过安装持久性后门实现，后门被视为攻击者的“访问点”（Access Point）。

常见持久性方法：
安装 Web Shell：攻击者在 Web 服务器上部署 Web Shell 恶意脚本，常见格式如 .php、.asp、.aspx、.jsp，利用其维持长期访问。由于其文件形式简单且伪装为普通文件，Web Shell 往往难以被检测。

安装后门程序：通过工具如 Meterpreter Framework 的交互式载荷，攻击者可以在目标设备上安装后门，远程控制系统并执行恶意代码。

创建或修改 Windows 服务：攻击者通过创建或修改服务来定期运行恶意脚本。使用工具：sc.exe 创建、启动、停止服务，reg 修改注册表配置。使用合法服务名称掩盖恶意载荷。

修改注册表或启动文件夹：攻击者在注册表的“运行键”或系统启动文件夹中添加入口，使恶意载荷在每次用户登录时自动执行。注册表可为单用户或全局路径，启动文件夹可影响所有账户登录。

攻击者还可以篡改文件的时间戳（Timestomping），修改时间、访问时间等伪装恶意文件为合法程序，避免取证检测。

指挥与控制（Command & Control）

建立持久性后，攻击者通过恶意软件创建 C2 通道，实现对受害者设备的远程控制。C2 通道 / C&C 或信标是指恶意软件与 C2 服务器之间的通信路径，用于传输指令和数据。被感染主机会定期向 C2 服务器发送请求，“信标”因此得名。

攻击者通过被感染设备主动与外部 C2 服务器通信，建立控制路径。一旦连接建立，攻击者即可完全接管受害者机器。早期攻击者常使用 IRC 作为 C2 通道，但现代安全工具能轻松检测 IRC 流量，因此不再广泛使用。现代攻击者更倾向于使用更隐蔽的通信方式。

常见 C2 通道类型：
HTTP 和 HTTPS：利用合法流量掩盖恶意通信，端口常为 80 或 443。能有效绕过防火墙和安全检测。
DNS 隧道（DNS Tunneling）：被感染设备通过发送 DNS 请求与攻击者控制的 DNS 服务器通信。此方法隐蔽性强，难以检测。

C2 服务器可能直接由攻击者掌控，也可能是另一个已被攻陷的主机。

目标行动（Actions on Objectives）

在完成前六个阶段后，攻击者进入最终目标阶段，开始执行具体行动。此阶段常见的攻击行为包括：

收集用户凭据：提取用户登录信息，以便进一步攻击系统或扩展访问权限。
权限提升：利用配置错误，从普通用户权限升级为高级权限。例如，获取域管理员权限。
内部侦察：与内部软件交互，查找系统中的漏洞和弱点，为后续行动做准备。
横向移动：在网络中移动，攻陷更多设备或服务器，扩展控制范围。
数据收集与外泄：转移公司敏感数据到外部服务器，窃取机密信息。
删除备份和卷影副本：卷影副本是微软提供的一项快照技术，用于备份文件或磁盘数据。删除备份文件和卷影副本（Shadow Copy），防止数据恢复。
覆盖或破坏数据：用垃圾数据覆盖文件，或直接损坏文件，使其无法恢复。
