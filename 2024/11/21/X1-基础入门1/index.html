域名差异 - 主站 & 分站 & 端口站 & 子站

主站

www.cyphersun.com 1.1.1.1

分站 / 子域名

blog.cyphersun.com 1.1.1.1 或 2.2.2.2

端口站

www.cyphersun.com:80 PHP
www.cyphersun.com:8080 Java

目录站

www.cyphersun.com 博客 PHP
www.cyphersun.com/bbs/ 论坛 Java

子站

123.blog.cyphersun.com

渗透目标和方法

域名与 IP 收集：确定目标主域名及子域名的所有 IP 地址
目录扫描：使用目录扫描工具识别公开的敏感目录和服务
子域名收集：利用工具发现子域名
端口扫描：使用工具扫描开放端口及其对应服务

源码差异 - 结构 & 语言 & 框架 & 闭源 & 加密

源码目录结构

后台目录：后台管理功能模块
文件目录：静态资源，如图片
逻辑目录：业务逻辑模块
前端目录：前端页面模板和资源
数据目录：数据库或数据文件存储
配置文件：系统或数据库的配置文件

源码开发语言

后端：ASP、ASPX、PHP、Java、Python、Go 等
前端：Javascript、HTML、CSS 等

语言开发框架

PHP：ThinkPHP、Laravel、YII、CodeIgniter、CakePHP、Zend 等
Java：Spring、MyBatis、Hibernate、Struts2、Spring Boot 等
Python：Django、Flask、Bottle、Turbobars、Tornado、Web2py 等
Javascript：Vue.js、Node.js、Bootstrap、jQuery、Angular 等

开源、闭源及加密类型

开源：如 Zblog、WordPress 等
闭源：如淘宝、百度内部的代码
加密源码：如通达 OA 等

数据差异 - 本地数据 & 分离数据 & 云数据库

数据库核心思想

通过外部连接实现源码与数据库的数据交互

数据库类型

关系型：Access、MySQL、SQL Server、Oracle、DB2、PostgreSQL
非关系型：Redis、MongoDB

本地数据库：源码和数据库部署在同一台服务器上

网站服务器：www.cyphersun.com 1.1.1.1
数据库服务器：1.1.1.1

分离数据库：源码和数据库部署在不同服务器上

网站服务器：www.cyphersun.com 1.1.1.1
数据库服务器：2.2.2.2

云数据库：使用云数据库，使用白名单访问权限以提高安全性，如 RDS

网站服务器：www.cyphersun.com 1.1.1.1
云数据库：3.3.3.3，仅允许来自1.1.1.1的访问

场景例子：

背景：
目标网站：www.cyphersun.com 1.1.1.1
数据库：云数据库，仅允许 IP 1.1.1.1 的连接

现状：
已通过 webshell 获取了目标网站的权限
数据库配置文件中包含了数据库的连接地址、账号、密码
由于本地 IP 不是 1.1.1.1，直接通过本地机器连接数据库会被拒绝

问题分析：
云数据库通过 IP 白名单限制，仅允许白名单 IP 1.1.1.1 访问，即使获取到数据库账号密码，也无法从本地直接连接数据库

解决：
只能让 1.1.1.1 去连接数据库

1.上传一个数据库连接脚本到网站上，访问脚本去连接数据库
2.利用网站权限上传代理流量工具，做数据中转的操作去连接数据库

平台差异 - 中间件类型 & 系统类型 & 容器类型

系统类型

Windows：如 Windows Server
Linux：如 CentOS、Ubuntu、Debian
MacOS：用于开发或特定部署环境

容器类型

容器技术：Docker、Kubernetes（K8s）
虚拟化技术：VMware、VirtualBox、Hyper-V

中间件类型

Web 服务器：Apache、Nginx、IIS、lighttpd
应用服务器：Tomcat、Jboss、WebLogic、WebSphere、Jetty

解析差异 - URL 路由 & 绝对相对路径 & 格式权限

URL 路由

静态路由：URL 与文件系统直接映射，路径明确。
www.cyphersun.com/admin/login.php = 源码/admin/login.php

MVC 模型：使用框架路由规则，URL 不对应物理文件路径
www.cyphersun.com/admin/login.php ≠ 源码/admin/login.php
MVC 框架通过路由解析逻辑映射到控制器和视图，如 Laravel、Spring

代码审计差异：
非 MVC 模型：直接审查 URL 对应的物理文件路径及逻辑代码
MVC 模型：需理解框架路由规则，定位路由规则与控制器的映射关系，再审查相关逻辑

相对路径与绝对路径

相对路径：基于当前目录进行定位
../../ 表示返回上两级目录

绝对路径：文件系统中的完整路径
D:\MYOA\webroot\attachment

路径解析：
D:\MYOA\webroot\attachment../../ → D:\MYOA
D:\MYOA\webroot\attachment../../../D:\aDrive → D:\aDrive

格式权限

后门解析格式，代码正常执行，脚本执行权限等
