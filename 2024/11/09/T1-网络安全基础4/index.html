侦察

从攻击者的角度看，侦察是发现并收集目标系统及受害者信息的过程，是攻击规划的首要阶段。攻击者通过侦察来了解目标系统的细节，识别潜在的攻击点。

公开来源情报（OSINT）是侦察的一部分，攻击者通常首先通过 OSINT 收集公司及员工的公开信息，此信息帮助攻击者确定最佳攻击目标。对于不知道要攻击哪个公司的攻击者来说，OSINT 使他们能够筛选和分析潜在目标。

电子邮件收集是攻击者在侦察阶段的常用方法之一，从公共、付费或免费服务中获取电子邮件地址。攻击者使用这些地址开展网络钓鱼攻击，窃取敏感数据。常见的侦察工具包括：

theHarvester：可收集电子邮件、姓名、子域名、IP 地址和 URL 等信息。
Hunter.io：获取与特定域名相关的联系信息。
OSINT Framework：提供基于不同类别的 OSINT 工具集合，支持多种侦察需求。

攻击者还利用社交媒体平台，收集目标公司或特定受害者的信息。

武器化

在完成侦察阶段后，攻击者会进入武器化阶段，创建一个用于投递的恶意有效载荷。通常，攻击者会选择不直接与受害者互动，而是利用武器生成器将恶意软件与漏洞利用结合生成有效载荷。大多数攻击者会使用自动化工具生成恶意软件，或从暗网获取现成的恶意代码；而更高级的攻击者或国家支持的 APT 组织则编写定制化恶意软件，使其具备规避检测的能力。

恶意软件（Malware）：设计用于损害、破坏或未经授权访问计算机的程序或软件。
漏洞利用（Exploit）：利用应用程序或系统漏洞的程序或代码。
有效载荷（Payload）：攻击者在系统上运行的恶意代码。

在武器化阶段，攻击者可能采取以下方式：
创建恶意 Office 文档：生成包含恶意宏或 VBA 脚本的 Microsoft Office 文档。
制作感染载体：生成复杂的恶意载荷或蠕虫病毒，植入 USB 驱动器并在公共场合分发。
选择指挥与控制（C2）：在受害者计算机上执行命令或传递更多有效载荷。
植入后门（Backdoor）：绕过安全机制，获取目标系统的持续访问权限。

投递

投递阶段是选择传递有效载荷或恶意软件的方法。常见的投递选项包括：

钓鱼邮件：在完成侦察并确定目标后，攻击者会向特定个人或公司多位员工发送带有恶意代码的邮件。邮件中包含有效载荷或恶意软件。

感染型 USB 驱动器分发：在公共场所分发带有恶意软件的 USB 驱动器，攻击者可能通过伪装成客户，将印有公司 Logo 的 USB 设备作为礼物寄送给公司，以实现 USB 投放攻击。

水坑攻击：有针对性的攻击针对特定人群。攻击者破坏受害者常访问的合法网站，将其重定向到恶意网站。通常会通过发送“无害”邮件，引导受害者点击恶意链接，从而达到攻击目的。受害者访问该网站时，可能会不经意间下载恶意软件或应用程序，这种攻击被称为“驱动下载”（Drive-by Download）。

漏洞利用

为了获得系统访问权限，攻击者通常利用系统的漏洞。在成功访问系统后，攻击者可以通过利用软件、系统或服务器的漏洞来提升权限，或在网络中横向移动。横向移动是指攻击者在初始入侵后进一步深入网络，以获取敏感数据的技术。

在此阶段，攻击者还可能使用零 day 漏洞利用。零 day 漏洞是尚未被公开或修补的漏洞，暴露了软件或硬件的弱点，可能在问题被发现前造成严重影响，且初期难以检测。

常见的漏洞利用方式:
受害者打开恶意电子邮件附件或点击恶意链接，触发漏洞利用
利用零 day 漏洞
利用软件、硬件漏洞，或利用人性弱点
触发服务器端的漏洞利用

安装

在成功获得系统访问权限后，攻击者通常会希望在连接中断、初始访问被移除或系统更新后，仍能保持对系统的访问。这时，攻击者需要安装持久性后门，以便持续访问被攻陷的系统。

实现持久性后门的方法:
在 Web 服务器上安装 Web Shell：Web Shell 是一种恶意脚本，使用 ASP、PHP 或 JSP 等语言编写，允许攻击者持续访问系统。Web Shell 因简单且文件格式不易检测，可能被误认为是正常文件。

在受害者机器上安装后门：攻击者可以使用工具在目标计算机上安装后门。

创建或修改 Windows 服务：攻击者可以创建或修改 Windows 服务，使恶意脚本或载荷定期执行，作为持久性的一部分。攻击者可使用 sc.exe 和 Reg 工具修改服务配置，并伪装服务名称，看似合法。

在注册表“运行键”或启动文件夹中添加恶意载荷：通过在这些位置添加条目，恶意载荷会在每次用户登录时自动执行。启动文件夹分为用户级和系统级，无论谁登录，系统级启动文件夹中的内容都会运行。

攻击者还可能使用时间戳篡改（Timestomping）技术来规避取证检测，使恶意软件的时间戳看起来与合法程序一致。

指挥与控制

在获得持久性并成功执行恶意软件后，攻击者通过指挥与控制（C2）通道实现对受害者计算机的远程控制。这种恶意通信方式也称为 C&C 或 C2 信标（C2 Beaconing），即受感染主机与 C2 服务器持续通信，定期向攻击者“信标”位置。

一旦被攻陷的终端与外部 C2 服务器建立连接，攻击者即可完全控制受害者计算机。

现代常见的C2通道:
HTTP（端口80）和 HTTPS（端口443）：恶意流量混入正常流量中，使攻击者更易绕过防火墙检测
域名服务器（DNS）：受感染的机器持续向攻击者控制的 DNS 服务器发送请求，这种通信方式称为 DNS 隧道（DNS Tunneling）

C2 基础设施的拥有者可以是攻击者自己，也可以是另一个被攻陷的主机。

目标行动（数据外泄）

在经历前六个攻击阶段后，攻击者最终可以直接控制系统并实现攻击目标，可能采取以下行动：

收集用户的登录凭证
执行权限提升
进行内部侦察
横向移动至公司网络的其他系统
收集并外泄敏感数据
删除备份和影子副本
覆盖或破坏数据
