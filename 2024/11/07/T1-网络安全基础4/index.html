侦察

从攻击者的角度来看，侦察是收集有关系统和目标的基础信息的过程，是攻击者在进行进一步攻击前的规划阶段。

开源情报（OSINT） 是侦察的重要组成部分，也是攻击的第一步。攻击者利用 OSINT 收集公司及其员工的公开信息，锁定最佳攻击目标。

电子邮件收集 是常用的 OSINT 技术之一。攻击者通过公开、付费或免费的资源获取目标的电子邮件地址，可利用这些信息实施网络钓鱼攻击，窃取敏感数据：

theHarvester：收集电子邮件、姓名、子域、IP地址和 URL 等信息。
Hunter.io：查找与特定域名相关的联系信息。
OSINT Framework：提供各种 OSINT 工具集合，按类别组织。

攻击者利用社交媒体收集个人和公司信息。这些信息为定向网络钓鱼等社交工程攻击提供了支持。

武器化

在完成侦察后，攻击者进入武器化阶段，武器生成器将恶意软件和漏洞利用组合成一个可交付的有效负载。大多数攻击者使用自动化工具生成恶意软件或通过暗网购买。而更高级的攻击者或国家支持的高级持续性威胁（APT）组织则可能编写定制的恶意软件，以逃避检测。

关键术语：

恶意软件（Malware）：旨在破坏、干扰或未经授权访问计算机的程序。
漏洞利用程序（Exploit）：利用系统或应用漏洞的程序或代码。
有效负载（Payload）：在目标系统上执行的恶意代码。

在武器化阶段，攻击者可能采取的策略包括：

创建包含恶意宏或 VBA（Visual Basic for Applications）脚本的感染性 Office 文档。

制作恶意负载或复杂蠕虫病毒，并将其植入 USB 驱动器中，通过公共场所分发进行传播。

采用指挥与控制（C2）技术，以便在目标设备上远程执行命令或投放更多有效负载。

后门植入，绕过安全机制，获得对系统的持续访问。

投递

在投递阶段，攻击者选择一种方式将有效负载或恶意软件传送给目标：

钓鱼邮件：经过侦察确定目标后，攻击者精心编写恶意邮件，针对特定个人或公司内部的多人，邮件中包含有效负载或恶意软件。

分发感染的 USB 驱动器：攻击者可能在咖啡馆、停车场或街道上投放感染的 USB 驱动器，或将带有公司标志的恶意 USB 邮寄给目标公司，伪装成客户赠品。

水坑攻击：攻击者入侵目标群体常访问的网站，将访问者重定向到恶意网站。攻击者可能会发送带有恶意网址链接的邮件，诱使受害者点击。一旦受害者访问该网站，可能会无意间下载恶意软件或应用程序。这种“驱动式下载”攻击可能包括伪装成浏览器扩展的恶意弹出窗口，引导受害者下载。

漏洞利用

在获得系统访问权限后，攻击者会利用系统中的漏洞来提升权限或在网络中横向移动，即在获得初始访问权限后进一步进入网络，以获取敏感数据。

攻击者可能使用零日漏洞进行攻击。零日漏洞是指尚未公开的漏洞，暴露了软件或硬件中的安全问题，通常难以在初期检测到，可能导致复杂的安全风险。

以下是攻击者进行漏洞利用的示例：

受害者打开恶意电子邮件附件或点击恶意链接，触发漏洞利用。
使用零日漏洞对目标实施攻击。
利用软件、硬件或人为因素中的漏洞。
触发服务器漏洞以获得更高权限或访问更多资源。

安装

在武器化阶段，后门允许攻击者绕过安全措施并隐藏其访问权限，也称为访问点。

一旦获得系统访问权限，攻击者可能希望在连接丢失、初始访问被移除、或系统被修补后仍能重新进入系统。攻击者会安装持久性后门，确保未来能再次访问已攻陷的系统。

持久性后门的实现方式包括：

安装 Web Shell：攻击者在 Web 服务器上植入恶意脚本以维持访问。由于 Web Shell 的简单性和常见的文件格式，它们通常难以被检测，可能被误认为正常文件。

在受害者机器上安装后门

创建或修改 Windows 服务：攻击者通过创建或修改 Windows 服务，使恶意负载定期执行，确保持久性。工具可用于配置服务，攻击者还可伪装恶意负载为系统或合法软件相关服务名称。

在注册表运行键或启动文件夹中添加恶意负载：这使得恶意负载在每次用户登录时自动执行。注册表和启动文件夹包含单用户和系统范围的选项，系统范围的启动文件夹在任何用户登录时都会被检查。

攻击者还可能使用时间戳篡改技术修改文件的时间戳，使恶意软件看似合法，从而避开检查。

指挥与控制

在获得持久性并在受害者的机器上成功执行恶意软件后，攻击者通过该恶意软件建立指挥与控制（C2）通道，以远程控制和操作受害者的设备。这种通信方式也称为 C&C 或 C2 信标，是 C2 服务器与被感染主机上的恶意软件之间的持续通信。被感染主机会不断向 C2 服务器发送信号，这种行为称为“信标”。

一旦受害者设备与攻击者设置的外部服务器建立通信，攻击者便能完全控制受害者的机器。C2 通道包括：

HTTP（端口80）和 HTTPS（端口443）：将恶意流量与正常流量混合，使攻击者更易绕过防火墙。

域名服务器（DNS）：被感染的主机会向攻击者的 DNS 服务器发送 DNS 请求，这种 C2 通信方式称为 DNS 隧道。

C2 基础设施的控制者可能是攻击者或另一个被攻陷的主机。

达成目标（数据窃取）

经过前六个攻击阶段，最终实现了初始任务，攻击者可以进行以下操作：

收集用户凭证
权限提升：利用配置错误获取更高级的权限
内部侦察：与内部软件交互，寻找系统漏洞
横向移动：在公司网络中扩展控制范围
收集并窃取敏感数据
删除备份和影子副本：影子副本是微软的一项技术，用于创建文件或卷的快照
覆盖或破坏数据
