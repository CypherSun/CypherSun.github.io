服务器消息块协议（SMB）

服务器消息块协议 (SMB) 是用于在客户端和服务器之间共享文件、打印机、端口以及其他资源访问的通信协议。通过 SMB 协议，服务器可通过网络向客户端提供文件系统和资源的访问权限，客户端则可以访问服务器上的共享文件和打印机。

SMB 是一种请求-响应协议，通过在客户端和服务器之间交换消息来建立连接。客户端常通过 TCP/IP 协议与服务器连接

SMB 是如何工作的？

建立连接后，客户端通过发送 SMB 命令来访问服务器上的共享资源，执行文件的打开、读写等操作，所有这些文件系统操作都通过网络完成。

什么系统支持 SMB？

自 Windows 95 起，Microsoft Windows 操作系统就内置了 SMB 协议的客户端和服务器支持。对于 Unix 系统，Samba 是一个支持 SMB 协议的开源服务器，用于在 Unix 和 Windows 系统之间共享文件和资源。

SMB 枚举

枚举

枚举是指收集目标系统信息的过程种发现潜在的攻击面，从而有效地进行漏洞利用。攻击者可以收集诸如用户名、密码、网络信息、主机名、应用程序数据、服务等关键信息，避免在无效漏洞利用上浪费时间和资源。

SMB

服务器通常提供可连接的 SMB 共享驱动器，用户可以通过这些共享访问或传输文件。SMB 常是攻击者获取敏感信息的首要入口

端口扫描

枚举的第一步通常是端口扫描，以识别目标机器上开放的端口和运行的服务。帮助收集关于服务、应用程序、网络结构和操作系统的信息。

Enum4Linux

Enum4Linux 是一个用于在 SMB 协议上枚举 Windows 和 Linux 系统共享资源的工具。它整合了 Samba 工具包中的功能，能快速提取 SMB 相关信息

标签 功能
-U 获取用户列表
-M 获取机器列表
-N 获取名称列表（不同于 -U 和 -M）

Telnet

Telnet 是什么？

Telnet 是一种应用层协议，允许用户通过 Telnet 客户端连接到远程主机，并在该主机上执行命令。前提是远程主机正在运行 Telnet 服务器。建立连接后，客户端会作为虚拟终端，用户可以通过它与远程主机交互。

替代

Telnet 以明文传输所有数据，缺乏安全机制，已在许多应用中被 SSH 替代。

Telnet 如何工作？

用户通过 Telnet 客户端连接到服务器，并可在连接后使用特定命令在服务器上执行指令。

通过端口扫描来识别目标机器上运行的服务、应用程序、系统结构和操作系统，获取尽可能多的基础信息

Telnet 漏洞类型

文件传输协议（FTP）

文件传输协议（FTP）是什么？

文件传输协议 (FTP) 是一种用于在网络上远程传输文件的协议。它采用客户端-服务器模式，通过独立的命令和数据通道来实现高效的文件传输。

FTP如何工作？

典型的 FTP 会话使用两个通道：

命令通道（控制通道）：传输命令和命令的响应
数据通道：传输文件数据

FTP 基于客户端-服务器协议。客户端发起与服务器的连接，服务器验证提供的登录凭据，成功后开启会话。会话建立后，客户端可以在服务器上执行各种 FTP 命令。

主动模式 vs 被动模式

FTP 服务器可以支持主动（Active）和被动（Passive）模式，或同时支持两者。

在主动 FTP 连接中，客户端打开一个端口并进行监听，服务器主动连接该端口进行数据传输
在被动 FTP 连接中，服务器打开一个端口并被动监听，客户端连接到该端口以传输数据

将命令和数据划分到不同的通道，可在不等待数据传输完成的情况下向服务器发送命令。提高了大文件传输或慢速网络连接时的效率
