网络文件系统（NFS）

什么是 NFS？

网络文件系统（Network File System，NFS）允许系统通过网络与其他系统共享目录和文件。借助 NFS，用户和程序可以像访问本地文件一样访问远程系统上的文件。通过将服务器上的整个或部分文件系统挂载到客户端系统上来实现。被挂载的文件系统部分可以由客户端访问，并按照分配给每个文件的权限进行访问。

NFS 如何工作？

客户端请求将远程主机上的某个目录挂载到本地目录，就像挂载物理设备一样。挂载服务将通过 RPC 连接到相关的挂载守护进程。

服务器会检查用户是否有权限挂载请求的目录。服务器会返回一个文件句柄，用于唯一标识服务器上的每个文件和目录

如果有人希望通过 NFS 访问文件，会向服务器上的 NFS 守护进程（NFSD）发出一个 RPC 调用。该调用包含以下参数：

文件句柄
要访问的文件名
用户的用户 ID
用户的组 ID

这些参数用于确定对指定文件的访问权限，从而控制用户的读写权限。

NFS 运行在什么系统上？

使用 NFS 协议可在运行 Windows 及其他非 Windows 操作系统的计算机之间传输文件。

运行 Windows Server 的计算机可以作为其他非 Windows 客户端计算机的 NFS 文件服务器。NFS 也允许基于 Windows 的计算机访问存储在非 Windows NFS 服务器上的文件。

枚举 NFS

什么是枚举？

枚举建立与目标主机的主动连接，以发现系统中潜在的攻击向量的过程，同时进一步对系统利用。指导攻击的信息将来自此阶段。

需求

为了对 NFS 服务器和共享进行更高级的枚举需要 nfs-common，这是用于在本地计算机上与NFS共享进行交互的工具包。

NFS-Common

任何使用 NFS 的机器都需要安装此软件包。该软件包包含多个程序。主要关注 showmount 和 mount.nfs，从NFS共享中提取信息时，这两个工具最好用。安装 nfs-common，这个软件包在大多数Linux发行版中都默认提供

可使用

端口扫描

枚举的第一步是执行端口扫描，以尽可能多地了解目标机器的服务、开放端口和操作系统，可使用带有-A和-p-参数的nmap 深入扫描

挂载 NFS 共享

客户端系统需要一个目录来访问主机服务器在导出文件夹中共享的内容。可以在系统的任何位置创建此目录。创建挂载点后，可使用 mount 命令将 NFS 共享连接到计算机上的挂载点


NFS 利用

如果在某台机器上获得低权限的 shell并可以 NFS 共享，可通过它提升权限，前提是 NFS 的配置允许

什么是root_squash？

默认情况下，NFS 共享启用了 Root Squashing 功能，阻止任何连接到 NFS 共享的用户获取 NFS 卷的 root 权限。远程 root 用户连接后会被赋予 nfsnobody 用户身份，这是权限最低的本地用户。但如果关闭该功能，就可以创建带 SUID 位的文件，这样远程用户就能以 root 身份访问系统

SUID

带 SUID 位的文件以文件所有者/组的权限运行。文件会以 root 的权限运行，获取具有权限的 shell

方法

将文件上传到 NFS 共享，并控制文件的权限。上传 bash shell 可执行文件。通过 SSH 登录，执行该文件以获得 root shell

可执行文件

使用在之前获取的密钥，直接从目标机器上获取 bash 可执行文件

另一种方法是从 nmap 扫描中了解到服务器版本，并获取与服务器相同的标准的 bash 可执行文件

操作路径：

具体步骤以及获得 root shell 的过程：

NFS 访问 -> 获取低权限 shell -> 将 bash 可执行文件上传到 NFS 共享 -> 通 NFS 设置 SUID 权限，利用错误配置的 Root Squash -> 通过 SSH 登录 -> 执行带有 SUID 位的 bash 可执行文件 -> 获得 ROOT 权限


简单邮件传输协议（Simple Mail Transfer Protocol，SMTP）

什么是 SMTP？

SMTP 是简单邮件传输协议（Simple Mail Transfer Protocol，SMTP）的缩写，用于处理电子邮件的发送。为了支持电子邮件服务，需要一对协议 SMTP 和 POP/IMAP。它们共同作用，使用户能够发送外发邮件并接收入站邮件

SMTP 服务器的三个基本功能如下：

验证通过 SMTP 服务器发送邮件的用户身份
发送外发邮件
如果外发邮件无法送达，将邮件退回给发送者

在配置第三方电子邮件客户端中的新邮箱时会遇到 SMTP；因为在设置新邮件客户端时，通常需配置 SMTP 服务器，以便能够发送外发邮件。

POP 和 IMAP

邮局协议（Post Office Protocol，POP）和互联网邮件访问协议（Internet Message Access Protocol，IMAP）都是负责在客户端和邮件服务器之间传输电子邮件的协议。

两者的主要区别在于：
POP 采用更简单的方式，将邮件服务器中的收件箱下载到客户端
IMAP 则同步当前收件箱与服务器上的新邮件，下载任何新的邮件。

如果通过 IMAP 在一台设备上对收件箱进行更改，另一台设备同步收件箱时这些更改也会保持。POP/IMAP 服务器负责执行这个过程。

SMTP 是如何工作的？

电子邮件的传递过程类似于物理邮件的投递。用户提供电子邮件，通过服务将其送达收件人的收件箱。SMTP 服务器的角色类似于邮政分拣中心：邮件被发送到此服务器，然后由它转发至收件人

电子邮件从用户计算机到达收件人的过程如下：

1.邮件用户代理连接到用户域的 SMTP 服务器，启动 SMTP 握手。此连接通过 SMTP 通常为端口 25 进行。一旦连接建立并通过验证，SMTP 会话即开始

2.开始发送邮件。客户端首先将发送方和收件人的电子邮件地址、邮件正文以及任何附件提交给服务器。

3.SMTP 服务器会检查收件人和发送方的域名是否相同

4.发送方的 SMTP 服务器会与收件方的 SMTP 服务器建立连接，将邮件中继传输过去。如果收件方的服务器无法访问或不可用，邮件会进入 SMTP 队列

5.收件方的 SMTP 服务器会验证收到的邮件，通过检查域名和用户名是否识别成功来完成验证。服务器验证后，会将邮件转发至 POP 或 IMAP 服务器

6。电子邮件会显示在收件人的收件箱中

SMTP 运行在什么系统上？

Windows 服务器平台上可以轻松获得 SMTP 服务器软件， SMTP 的其他版本也可在 Linux 上运行

从 SMTP 枚举用户

SMTP 服务提供了两个用于枚举用户的内部命令：VRFY验证有效用户的名称和 EXPN显示用户别名及邮件列表的实际地址。通过这些 SMTP 命令，可生成一份有效用户列表

Metasploit 名为 smtp_enum 的模块能够完成，使用该模块只需提供一个主机或主机范围，以及包含用户名的字典文件，即可进行用户枚举。

MySQL

什么是 MySQL？

MySQL 是一种基于结构化查询语言（SQL）的关系型数据库管理系统（RDBMS）。

数据库：
数据库是一个持久化的、有组织的结构化数据集合。

RDBMS：
RDBMS 是一种用于创建和管理基于关系模型的数据库的软件或服务。“关系型”意味着数据存储在表格中，每个表格通过“主键”或其他“键”因素与其他表格相关联。

SQL：
MySQL 是最受欢迎的关系型数据库管理系统软件之一的品牌名称。它使用客户端-服务器模型。使用了一种语言，即结构化查询语言（SQL）。

MySQL 是如何工作的？

作为一种关系型数据库管理系统（RDBMS），MySQL 由服务器和一些管理 MySQL 数据库的实用程序组成。

服务器处理所有数据库指令，比如创建、编辑和访问数据。它接收并管理这些请求，并通过 MySQL 协议与客户端通信。整个过程可以分为以下几个步骤：

MySQL 创建一个数据库用于存储和操作数据，并定义每个表之间的关系。
客户端通过 SQL 语句发出特定请求。
服务器响应客户端请求，返回所需的信息。

MySQL 可以运行在哪些系统上？

MySQL 可以在多种平台上运行，包括 Linux 和 Windows。它通常被用作许多大型网站的后端数据库，是 LAMP 技术栈的重要组成部分，其中包括：Linux、Apache、MySQL 和 PHP。

Hashes（哈希）：

哈希，简单来说，是一种加密算法的产物，用来将可变长度的输入转换为固定长度的输出。

在MySQL中，哈希可以以不同的方式使用。每个哈希都有一个唯一的ID，作为指向原始数据的指针。这种方式创建的索引比原始数据要小得多，使得数据能够更高效地被搜索和访问。
