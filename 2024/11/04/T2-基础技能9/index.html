网站的工作原理

当访问一个网站时，例如 Google Chrome 浏览器会向一个网页服务器发出请求，以获取正在访问页面的信息。服务器收到请求后，将所需的数据返回给浏览器，以便浏览器呈现页面内容。网页服务器实际上就是对方计算机，用来处理用户的请求并返回相应数据。

一个网站由两个主要部分组成：

前端（客户端）：即浏览器如何展示网站的部分。它负责网页的视觉效果、布局和交互，让用户可以直观地访问内容。

后端（服务器端）：位于服务器上，处理请求并返回数据。后端负责接收用户的操作请求，执行相关逻辑或数据处理，最终生成相应的内容。

当浏览器向网页服务器发出请求时，这一过程中还会涉及包括安全验证和数据传输加密等。基本流程为：浏览器发送请求到服务器，服务器返回数据，浏览器展示网页内容给用户。


HTML

网站主要通过以下三种技术创建：

超文本标记语言（HTML）：用于构建网站的结构。
层叠样式表（CSS）：用于美化网站，使其具备视觉效果。
JavaScript：用于实现复杂的互动功能，让页面更具交互性。

HTML 是构建网站的基础语言。HTML 页面的基本构成单元是元素/标签，这些标签告诉浏览器如何展示页面内容，所有网站的结构基本相似。

JavaScript

JavaScript（JS）是世界上最流行的编程语言之一，它为网页提供了交互功能。HTML 用于构建网页的结构和内容，而 JavaScript 则负责网页的行为和功能。没有 JavaScript，网页将保持静态，缺乏互动性。JS 可以动态更新页面内容，实现交互效果，例如在用户点击按钮时改变按钮样式或显示动画等。

敏感数据暴露

敏感数据暴露是指网站未妥善保护用户端的敏感信息，导致这些信息以明文形式在网站前端的源代码中暴露。攻击者可以通过查看页面源代码轻松获得这些数据。

网站由许多 HTML 元素/标签构建。开发人员有时可能会忘记移除测试期间的登录凭证、指向私密部分的隐藏链接或其他敏感信息，导致它们在 HTML 或 JavaScript 中泄露。这类信息一旦暴露，可能会被攻击者利用，从而进一步访问 Web 应用程序的不同部分。例如，HTML 注释中残留的临时登录凭证可能被滥用，用于访问应用程序的其他部分，甚至后台组件。

在对 Web 应用程序进行安全性评估时，首先应查看页面源代码，检查是否存在暴露的登录凭证、隐藏链接或其他敏感数据。这是识别潜在安全风险的重要一步。

HTML注入

HTML 注入是一种漏洞，当未过滤的用户输入直接显示在页面上时可能发生。如果网站未对用户输入进行清理过滤掉恶意内容，攻击者可以向易受攻击的网站注入 HTML 代码，从而改变页面的内容或行为。

用户输入往往会被用于其他前端和后端功能。如果不加清理就使用这些输入，网站可能会受到各种攻击。在 HTML 注入中，攻击者可以在页面上插入 HTML 或 JavaScript 代码，浏览器会将其渲染为实际页面内容，使攻击者能够控制页面的外观和功能。

例如，一个表单将“What’s your name”字段中的用户输入内容传递给 JavaScript 函数并输出到页面上。如果用户在该字段中输入 HTML 或 JavaScript 代码，浏览器会将这些代码直接渲染到页面上，导致页面被控制。例如，通过插入 <h1> 标签，用户的输入将被当作纯 HTML 显示。

安全的基本原则是永远不要信任用户输入。为了防止 HTML 注入，开发人员应在使用用户输入之前对其进行清理。
