什么是 DNS？

什么是 DNS？
DNS（域名系统）为我们提供了一种简单的方式，让我们在互联网上与设备通信时无需记住复杂的数字。就像每个房屋都有一个唯一的地址用于接收邮件一样，互联网上的每台计算机也有一个用于通信的唯一地址，称为 IP 地址。IP 地址的格式类似于 104.26.10.229，由 0 到 255 之间的四组数字组成，每组数字之间用句号分隔。当访问一个网站时，记住这组复杂的数字显然不方便，而这就是 DNS 的作用。通过 DNS，只需记住 tryhackme.com，而不必记住 104.26.10.229。

域名层级结构

域名层级结构
一张域名层级结构的示意图，最顶端是根域，向下分支为多个顶级域名，再进一步分支为二级域名。

顶级域名（TLD，Top-Level Domain）
顶级域名是域名的最右侧部分。例如，在 tryhackme.com 中，顶级域名是 .com。顶级域名分为两种：gTLD（通用顶级域名）和 ccTLD（国家代码顶级域名）。历史上，gTLD 用于指明域名的用途，例如 .com 表示商业用途，.org 表示组织机构，.edu 表示教育用途，.gov 表示政府机构；而 ccTLD 用于地理定位，例如 .ca 表示加拿大，.co.uk 表示英国等。由于需求激增，越来越多的 gTLD 被引入，包括 .online、.club、.website、.biz 等。点击此处可查看超过 2000 个 TLD 的完整列表。

二级域名
以 tryhackme.com 为例，.com 是顶级域名，而 tryhackme 是二级域名。在注册域名时，二级域名加上顶级域名的字符总数限制为 63 个字符，并且只能使用 a-z、0-9 和短横线（短横线不能出现在开头或结尾，且不能连续出现）。

子域名
子域名位于二级域名的左侧，并用句号分隔。例如，在 admin.tryhackme.com 中，admin 是子域名。子域名的命名限制与二级域名相同，最多为 63 个字符，只能使用 a-z、0-9 和短横线（短横线不能出现在开头或结尾，且不能连续出现）。可以使用多个句号分隔来创建更长的名称，例如 jupiter.servers.tryhackme.com。但总长度必须在 253 个字符以内。对于域名，可以创建的子域名数量没有限制。


记录类型

DNS 记录类型
DNS 不仅仅用于网站解析，它还包含多种类型的 DNS 记录。以下是一些最常见的记录类型。

A 记录
该记录解析为 IPv4 地址，例如 104.26.10.229。

AAAA 记录
该记录解析为 IPv6 地址，例如 2606:4700:20::681a:be5。

CNAME 记录
该记录解析为另一个域名。例如，TryHackMe 的在线商店子域名 store.tryhackme.com 返回一个 CNAME 记录 shops.shopify.com。随后会对 shops.shopify.com 发起另一个 DNS 请求，以确定其 IP 地址。

MX 记录
该记录解析为处理邮件的服务器地址。例如，tryhackme.com 的 MX 记录可能会返回 alt1.aspmx.l.google.com。MX 记录还包含优先级标记，用于告知客户端按照顺序尝试连接服务器，以便在主服务器不可用时将邮件发送到备用服务器。

TXT 记录
TXT 记录是自由文本字段，可以存储任意文本数据。TXT 记录用途广泛，常见用途包括列出有权代表该域名发送邮件的服务器（这有助于防止垃圾邮件和邮件伪造）。它们还可用于在注册第三方服务时验证域名的所有权。

发起请求

当发起一个 DNS 请求时会发生什么
一张示意图展示了文本中描述的流程。

当请求一个域名时，电脑首先会检查本地缓存，查看是否近期已查询过该地址；如果没有缓存，则会向递归 DNS 服务器发起请求。

递归 DNS 服务器通常由互联网服务提供商（ISP）提供，但也可以选择自己的 DNS 服务器。该服务器也有一个本地缓存，保存了近期查询的域名记录。如果本地找到结果，服务器会将其返回给电脑，查询过程在此结束（这种情况在 Google、Facebook、Twitter 等热门服务中很常见）。如果请求在本地找不到，接下来的查询将从互联网的根 DNS 服务器开始。

根服务器作为互联网 DNS 的主干网络，其任务是将请求重定向到合适的顶级域名（TLD）服务器。例如，如果请求 www.tryhackme.com，根服务器会识别出 .com 顶级域名，并将请求指向处理 .com 地址的 TLD 服务器。

TLD 服务器保存关于如何找到权威服务器的记录，而权威服务器会对 DNS 请求做出最终响应。权威服务器通常也称为该域的名称服务器。例如，tryhackme.com 的名称服务器是 kip.ns.cloudflare.com 和 uma.ns.cloudflare.com。通常一个域名会有多个名称服务器，以便在其中一个故障时提供备份。

权威 DNS 服务器负责存储特定域名的 DNS 记录，并且是更新域名 DNS 记录的地方。根据记录类型，DNS 记录会返回到递归 DNS 服务器，递归服务器会缓存该记录以供将来使用，并将结果传回最初发起请求的客户端。所有 DNS 记录都有一个 TTL（生存时间）值。该值以秒为单位，表示本地应保存响应的时间，过期后需要重新查询。缓存可以减少每次与服务器通信时发起 DNS 请求的频率。
