什么是 DNS？

域名系统（DNS）提供了一种能够在互联网上与设备通信，而无需记住复杂的数字 IP 地址的方法。互联网上的每台计算机都有一个独特的 IP 地址用于通信。IP 地址的格式通常是类似于 1.1.1.1 的数字组合，由四组数字构成，每组范围在 0 到 255 之间，并用点分隔。当访问一个网站时，通过 DNS，只需记住类似 baidu.com 的域名，而不必记住数字 IP 地址。

域名层次结构

域名层次结构中根域名位于最上方，向下分支为多个顶级域名（TLD），再进一步分支为二级域名。

顶级域名（TLD）

顶级域名（TLD）是域名最右侧的部分。例如，在 baidu.com 中，TLD 是 .com。顶级域名分为两种类型：通用顶级域名（gTLD）和国家代码顶级域名（ccTLD）。

gTLD 指示域名的用途，如 .com 表示商业用途，.org 表示组织机构，.edu 表示教育机构，.gov 表示政府机构。也新增了许多 gTLD，例如 .news、.store、.blog、.fun 等。
ccTLD 用于地理位置区分，如 .cn 表示中国，.us 表示美国。

二级域名

在 baidu.com 中，.com 是 TLD，而 baidu 是二级域名。注册域名时，二级域名的长度限制为 63 个字符，字符仅支持 a-z、0-9 和连字符。

子域名

子域名位于二级域名的左侧，通过句点分隔。例如，在 tieba.baidu.com 中，tieba 是子域名。子域名的创建规则与二级域名相同，长度限制为 63 个字符，仅支持 a-z、0-9 和连字符。但可以通过多个点分隔子域名来创建更长的名称，如 c.tieba.baidu.com，但域名的总长度必须控制在 253 个字符以内，且子域名的数量没有限制。


记录类型

DNS 记录类型

DNS 还包括多种记录类型。以下是一些常见的 DNS 记录类型：

A 记录
A 记录解析域名为 IPv4 地址，例如 1.1.1.1

AAAA 记录
AAAA 记录解析域名为 IPv6 地址，例如 2024:1104:10::22ab

CNAME 记录
CNAME 记录将域名指向另一个域名。例如，baidu 贴吧使用子域名 www.tieba.com，它的 CNAME 记录可能指向 tieba.baidu.com。然后对 tieba.baidu.com 发起另一个 DNS 请求以获取实际 IP 地址。

MX 记录
MX 记录用于指定负责处理邮件的服务器地址。例如，google.com 的 MX 记录可能返回 aspmx.l.google.com。MX 记录还包含优先级标记，指示客户端按照优先级顺序选择邮件服务器，这在主服务器故障时可将邮件重定向到备用服务器。

TXT 记录
TXT 记录是自由文本字段，可存储任意文本信息，常用于列出有权代表该域发送邮件的服务器，帮助防止垃圾邮件和伪造邮件。TXT 记录也用于域名所有权验证，例如在注册第三方服务时。

发起请求

当发起 DNS 请求时会发生什么

当请求一个域名时，计算机会先检查本地缓存，查看是否近期查询过该地址。如果有缓存，直接使用缓存记录；否则，会向递归 DNS 服务器发起请求。

递归 DNS 服务器通常由互联网服务提供商（ISP）提供，但用户也可以选择其他递归服务器。该服务器也有一个本地缓存。如果缓存中存在结果，则直接返回给计算机，结束查询。若未找到结果，递归服务器会开始查询过程，从互联网的根 DNS 服务器开始。

根服务器是 DNS 的核心，将请求引导至正确的顶级域名（TLD）服务器。例如，查询 www.baidu.com 时，根服务器识别顶级域名 .com，并将请求引导至管理 .com 的 TLD 服务器。

TLD 服务器保存指向权威服务器的记录。权威服务器/名称服务器是负责特定域名的服务器，例如 baidu.com 的名称服务器为 ns1.baidu.com 到 ns6.baidu.com。域名会有多个名称服务器，确保主服务器故障时有备选。

权威 DNS 服务器存储并维护特定域名的 DNS 记录。根据请求的记录类型，权威服务器将结果传回递归 DNS 服务器，递归服务器缓存结果以备将来使用，并最终将结果返回给最初发起请求的客户端。所有 DNS 记录都有一个生存时间（TTL）值，表示该记录在缓存中保存以秒为单位的时间。TTL 到期后，需重新发起查询。缓存机制减少了重复请求，提高了通信效率。
