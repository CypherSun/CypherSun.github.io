综合整理

当在浏览器中请求网页时，后台会进行一系列操作。首先计算机需要通过 DNS 系统获取目标服务器的 IP 地址。然后通过 HTTP 协议与该服务器建立通信，服务器会返回网页所需的内容，包括 HTML、JavaScript、CSS、图片等。浏览器接收到内容后，将其格式化并展示为网站。

例如缓存、内容分发网络（CDN）等组件可以提高网络效率并提供额外的功能，从而确保页面加载更快、体验更流畅。

其他组件

负载均衡器

当一个网站流量增大或需要高可用性支持时，单一的 Web 服务器可能无法满足需求。负载均衡器为高流量网站提供了两个关键功能：负载分担和故障转移。请求进入带有负载均衡器的网站时，负载均衡器会先接收请求，然后根据算法将其转发到最合适的服务器。常见的算法包括轮询法进行依次分配请求和加权法将请求分配给最空闲的服务器。负载均衡器会定期对每台服务器进行健康检查，若服务器无响应，则暂停向该服务器发送流量，直到其恢复。

内容分发网络（CDN）

CDN 是减轻繁忙网站流量的有效方法。它通过全球多个服务器托管网站的静态文件，如 JavaScript、CSS、图片、视频等，当用户请求这些资源时，CDN 会将请求定向至离用户最近的服务器，从而减少延迟并加速加载。

数据库

网站通常需要数据库来存储用户信息和内容。Web 服务器通过数据库进行数据存储和检索，数据库类型多样，从简单的文本文件到多台服务器的集群，能够提供快速且可靠的数据管理。常用的数据库系统包括 MySQL、MSSQL、MongoDB、Postgres 等，每种数据库系统具有独特的特性和用途。

Web 应用防火墙（WAF）

WAF 位于 Web 请求和 Web 服务器之间，保护服务器免受攻击或恶意流量的影响。WAF 通过分析请求内容，检测常见的攻击模式，如 SQL 注入、XSS 攻击等，并通过速率限制控制请求频率，避免异常流量对服务器造成冲击。如果检测到可疑请求，WAF 会将其拦截，防止其到达 Web 服务器。


Web 服务器的工作原理

Web 服务器

Web 服务器是一种软件，用于监听传入的连接请求并通过 HTTP 协议向客户端传送网页内容。常见的 Web 服务器软件包括 Apache、Nginx、IIS 和 NodeJS。Web 服务器会从根目录提供文件，该目录的位置在服务器设置中定义。例如，在 Linux 系统，Nginx 和 Apache 的默认根目录是 /var/www/html。在 Windows 系统，IIS 的默认根目录是 C:\inetpub\wwwroot。当请求 http://www.example.com/wechat.jpg 时，服务器会从本地路径 /var/www/html/wechat.jpg 提供该文件。

虚拟主机

Web 服务器能够托管多个不同域名的网站，这种功能称为虚拟主机。服务器会检查 HTTP 请求头中的主机名并与虚拟主机配置匹配，找到匹配项后，返回对应的网站。没有匹配项则返回默认网站。虚拟主机的根目录可以指向硬盘上的不同位置，例如 cyphersun.com 可以映射到 /var/www/cypher1，而 cyphersun.fun 映射到 /var/www/cypher2。理论上 Web 服务器可以托管无限个网站。

静态内容与动态内容

静态内容指的是不会随请求改变的内容，例如图片、JavaScript、CSS 文件，或固定的 HTML 文件。这些文件直接由 Web 服务器提供，无需进一步处理。动态内容则会根据请求而改变，例如博客主页会随着新文章发布而更新，或搜索页面会根据关键词返回不同结果。动态内容的生成在后端完成，使用脚本语言或编程语言生成用户不可见的 HTML。

脚本语言与后端语言

后端语言用于处理服务器端的交互和逻辑，实现与用户的互动。常见的后端语言包括 PHP、Python、Ruby、NodeJS、Perl 等。后端代码可以与数据库交互、调用外部服务、处理用户数据等。例如访问 http://example.com/index.php?name=cyphersun
