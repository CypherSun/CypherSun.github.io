介绍 LAN 拓扑

局域网 (LAN) 拓扑

多年来，人们一直在试验和实施各种网络设计。当谈到网络，提到“拓扑”一词时，实际上指的是现有网络的设计或外观。下面关于这些拓扑的优缺点。

星型拓扑

星型拓扑的主要前提是设备通过交换机或集线器等中央网络设备单独连接。这种拓扑是当今最常见的拓扑，因为它具有可靠性和可扩展性 - 尽管成本较高。

任何发送到此拓扑中的设备的信息都通过其连接的中央设备发送。下面关于此拓扑的一些优点和缺点：

由于此拓扑需要更多布线和购买专用网络设备，因此它比任何其他拓扑都更昂贵。然而，尽管增加了成本，但这确实提供了一些显着的优势。例如，这种拓扑本质上更具可扩展性，这意味着随着网络需求的增加，可以非常轻松地添加更多设备。

但是网络规模越大，需要的维护就越多，才能保持网络正常运行。对维护的依赖性增加也会使故障排除变得更加困难。此外，星型拓扑仍然容易发生故障 - 尽管发生故障的几率有所降低。例如，如果连接设备的集中式硬件发生故障，这些设备将不再能够发送或接收数据。但这些集中式硬件设备通常非常坚固。

总线拓扑

这种连接依赖于一条称为主干电缆的连接。这种拓扑结构类似于树上的叶子，因为设备从电缆上的分叉处开始。

由于发往每台设备的所有数据都通过同一条电缆传输，因此如果拓扑中的设备同时请求数据，则很快就会变得缓慢并出现瓶颈。这种瓶颈还会导致非常困难的故障排除，因为很快就很难确定哪台设备在数据沿同一条路径传输时遇到了问题。

然而，总线拓扑是更容易和更具成本效益的拓扑之一，因为它的成本较低，例如用于连接这些设备的电缆或专用网络设备。

总线拓扑的另一个缺点是在发生故障时几乎没有冗余。这个缺点是因为主干电缆上有一个单点故障。如果这条电缆断了，设备就无法再沿总线接收或发送数据。

环形拓扑

环形拓扑也称为令牌拓扑具有一些相似之处。计算机等设备直接相互连接以形成环路，这意味着几乎不需要布线，并且对星型拓扑中专用硬件的依赖性较低。

环形拓扑的工作原理是将数据发送到环路中，直到到达目标设备，然后使用环路上的其他设备转发数据。在这种拓扑中，如果设备没有数据要发送，它只会发送从其他设备收到的数据。如果设备恰好有数据要发送，它会先发送自己的数据，然后再发送来自其他设备的数据。

由于数据在这种拓扑结构中只能单向传输，因此很容易排除任何故障。然而，这是一把双刃剑，因为它不是数据传输的有效方式，因为它可能必须先访问许多设备才能到达目标设备。

环形拓扑结构不会像总线拓扑结构一样容易出现瓶颈，因为大量流量不会同时通过网络。然而，这种拓扑结构的设计意味着，电缆切断或设备损坏等故障将导致整个网络中断。

什么是 Switch？

交换机是网络中的专用设备，旨在通过以太网聚合多台其他设备，例如计算机、打印机或任何其他具有网络功能的设备。这些不同的设备插入交换机的端口。交换机通常用于大型网络，例如企业、学校或类似规模的网络，这些网络中有许多设备要连接到网络。交换机可以连接大量设备，因为它们有 4、8、16、24、32 和 64 个端口供设备插入。

交换机比集线器/中继器低端同类设备效率高得多。交换机会跟踪哪个设备连接到哪个端口。当它们收到数据包时，它不会像集线器那样将该数据包转发到每个端口，而是直接将其发送到预期目标，从而减少网络流量。

交换机和路由器都可以相互连接。通过增加多条数据路径，可以提高网络的冗余度或可靠性。如果一条路径出现故障，可以使用另一条路径。虽然这可能会降低网络的整体性能，因为数据包需要更长的时间才能传输，但不会出现停机时间。

什么是路由器？

路由器的作用是连接网络并在网络间传递数据。它通过路由来实现这一点，因此得名路由器。

路由是数据在网络中传输的过程的标签。路由涉及在网络之间创建路径，以便成功传输数据。

当设备通过多条路径连接时，路由很有用。

子网划分入门

正如之前在整个模块中讨论的那样，网络有各种形状和大小 - 从小到大。子网划分是指将网络划分为更小的微型网络。；例如朋友过生日为朋友切蛋糕。蛋糕只有一块，但每个人都想分一小块。子网划分就是决定谁分哪一块，并保留这一块。

以一家企业为例；将拥有不同的部门，例如：
会计
金融
人力资源

虽然知道在现实生活中应该将信息发送到正确的部门，网络也同样需要知道。网络管理员使用子网划分来对网络的特定部分进行分类和分配，以反映这一点。

子网划分是通过划分网络中可容纳的主机数量来实现的，用一个称为子网掩码的数字表示。

IP 地址由四个部分组成，称为八位字节。子网掩码也是如此，它也表示为四个字节，32 位的数字，范围从 0 到 255。

子网以三种不同的方式使用 IP 地址：
识别网络地址
识别主机地址
识别默认网关

将这三者分开，以了解它们的用途，如下表所示：

类型 目的 解释 例子
网络地址 标识实际网络的起点并用于识别网络的存在 例如，IP 地址为 192.168.1.100 的设备将位于 192.168.1.0 标识的网络上 192.168.1.0
主机地址 用于标识子网上的设备 例如，设备的网络地址为 192.168.1.1 192.168.1.100

现在在家庭等小型网络中，将位于一个子网中，因为不太可能需要同时连接超过 254 台设备。

在企业和办公室等场所将拥有更多此类设备，例如个人电脑、打印机、摄像机和传感器，而子网划分正是在这些地方进行的。

子网划分提供了一系列好处，包括：
效率
安全
完全控制

关于子网划分的安全性。以街上的典型咖啡馆为例。这家咖啡馆将有两个网络：
一个用于员工、收银机和设施的其他设备
一个供公众使用的热点

通过子网划分，可以将这两个用例彼此分离，同时享受连接到例如 Internet 更大网络的好处。

地址解析协议

设备可以有两个标识符：一个 MAC 地址和一个 IP 地址，地址解析协议 (ARP) 是负责允许设备在网络上识别自己的技术。

简单来说，ARP 允许设备将其 MAC 地址与网络上的 IP 地址关联起来。网络上的每个设备都会保留与其他设备关联的 MAC 地址的日志。

当设备希望与另一台设备通信时，它们会向整个网络发送广播以搜索特定设备。设备可以使用 ARP 查找设备的 MAC 地址，以及物理标识符以进行通信。

ARP 如何工作？

网络中的每个设备都有一个用于存储信息的缓存。在 ARP 中，此缓存存储网络上其他设备的标识符。

为了将这两个标识符，IP 地址和 MAC 地址映射在一起，ARP 发送两种类型的消息：
ARP请求
ARP应答

发送 ARP 请求时，网络上会向其他设备广播一条消息，询问 “拥有此 IP 地址的 MAC 地址是什么？” 当其他设备收到该消息时，它们只有在拥有该 IP 地址时才会响应，并发送带有其 MAC 地址的 ARP 回复。请求设备然后可以记住此映射并将其存储在其 ARP 缓存中以供将来使用。

DHCP

IP 地址可以通过将其物理输入到设备中进行手动分配，或最常见的是服务器使用动态主机配置协议 (DHCP) 自动分配。当设备连接到网络时，如果尚未手动分配 IP 地址，它会发出DHCP 发现请求以查看网络上是否有任何 DHCP 服务器。然后， DHCP 服务器回复DHCP 提供设备可以使用的 IP 地址。然后，设备发送DHCP 请求回复确认它需要提供的 IP 地址，最后，DHCP 服务器发送DHCP 确认确认已完成，并且设备可以开始使用 IP 地址。
