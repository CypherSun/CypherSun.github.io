您可以发送信号来终止进程；有多种类型的信号与内核处理进程的“干净程度”相关。要终止命令，我们可以使用适当命名的kill 命令和我们希望终止的相关 PID。例如，要终止 PID 1337，我们将使用kill 1337。

进程如何启动？

我们先来谈谈命名空间。操作系统 ( OS )使用命名空间将计算机上可用的资源（如CPU、RAM和优先级）最终分配给进程。可以将其想象为将计算机分成几块——类似于蛋糕。该块中的进程将有权使用一定数量的计算能力，但这只是每个进程实际可用资源的一小部分。

命名空间对于安全性非常有用，因为它是一种将进程与其他进程隔离的方式——只有位于同一命名空间中的进程才能互相看到。

我们之前讨论过 PID 的工作原理，这里就介绍一下它的作用。ID 为 0 的进程是系统启动时启动的进程。此进程是 Ubuntu 上的系统 init，例如systemd，用于提供一种管理用户进程的方法，位于操作系统和用户之间。

例如，系统启动并初始化后，systemd是第一个启动的进程之一。我们想要启动的任何程序或软件都将作为systemd的子进程启动。这意味着它由systemd控制，但将作为自己的进程运行（尽管共享来自systemd的资源），以便我们更容易识别等等。

Linux中的后台和前台 简介

进程可以以两种状态运行：在后台和在前台。例如，您在终端中运行的命令（如“echo”）或类似命令将在终端的前台运行，因为它是唯一未被告知在后台运行的命令。“echo”就是一个很好的例子，因为 echo 的输出将在前台返回给您，但不会在后台返回 - 例如，请看下面的屏幕截图。

这里我们正在运行echo “Hi THM” ，我们期望输出像开始时一样返回给我们。但是在将&运算符添加到命令后，我们只获得了 echo 进程的 ID，而不是实际的输出——因为它在后台运行。

这对于复制文件等命令非常有用，因为这意味着我们可以在后台运行该命令并继续执行我们想要执行的任何其他命令（而不必等待文件复制完成）

维护您的系统：自动化

用户可能希望安排在系统启动后执行某个操作或任务。例如，运行命令、备份文件或启动您最喜欢的程序（如 Spotify 或 Google Chrome）。

我们将讨论该cron进程，但更具体地说，我们如何通过使用与它进行交互crontabs。 Crontab 是在启动期间启动的进程之一，负责促进和管理 cron 作业。

crontab 只是一个特殊的文件，其格式可被进程识别cron，并可逐步执行每行。crontab 需要 6 个特定值：

价值 描述
分钟 在什么时候执行
小时 在什么时间执行
DOM 在每月的哪一天执行
星期一 在一年中的哪个月执行
陶氏 在星期几执行
命令 将要执行的实际命令。

让我们以备份文件为例。您可能希望每 12 小时备份一次“cmnatic”的“文档”。我们将使用以下格式：

0 */12 * * * cp -R /home/cmnatic/Documents /var/backups/

crontabs 的一个有趣特性是它们还支持通配符或星号 ( *)。如果我们不想为该特定字段提供值，即我们不关心它在哪月、哪日或哪年执行 — — 只关心它每 12 小时执行一次，我们只需放置一个星号即可。

一开始这可能会让人感到困惑，这就是为什么有一些很棒的资源，例如在线“ Crontab Generator ”，它允许您使用友好的应用程序为您生成格式！以及网站“ Cron Guru ”！

可以使用 来编辑 Crontabs crontab -e，您可以选择一个编辑器（例如 Nano）来编辑您的 crontab。

管理您的存储库（添加和删除）

通常我们使用 apt 命令将软件安装到我们的 Ubuntu 系统上。该apt命令是包管理软件（也称为 apt）的一部分。Apt 包含一整套工具，可让我们管理软件的包和源，并同时安装或删除软件。

添加存储库的一种方法是使用 add-apt-repository我们上面说明的命令，但我们将 逐步介绍如何手动添加和删除存储库。虽然您可以通过使用诸如 之类的软件包安装程序来安装软件dpkg，但 apt 的好处在于，每当我们更新系统时，包含我们添加的软件的存储库也会被检查是否有更新。

在这个例子中，我们将把文本编辑器 Sublime Text 作为存储库添加到我们的 Ubuntu 机器中，因为它不是默认 Ubuntu 存储库的一部分。添加软件时，我们下载的完整性由所谓的 GPG（Gnu Privacy Guard）密钥保证。这些密钥本质上是开发人员的安全检查，表示“这是我们的软件”。如果密钥与您的系统信任的密钥和开发人员使用的密钥不匹配，则不会下载该软件。

因此，首先，我们需要为 Sublime Text 3 的开发人员添加 GPG 密钥。（请注意，TryHackMe 实例无法访问互联网，因此我们不希望您将其添加到您部署的机器上，因为它会失败。）
