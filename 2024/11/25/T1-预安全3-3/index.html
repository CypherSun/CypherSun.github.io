网站如何运作

在本房间结束时，将了解网站是如何创建的，并将了解一些基本的安全问题。

当访问网站时，的浏览器（如 Safari 或 Google Chrome）会向网络服务器发出请求，询问有关正在访问的页面的信息。它会响应的浏览器用来显示页面的数据；网络服务器只是世界上某个地方处理的请求的专用计算机。

网站由两个主要部分组成：

前端（客户端）——的浏览器呈现网站的方式。
后端（服务器端）——处理的请求并返回响应的服务器。
的浏览器向 Web 服务器发出请求还涉及许多其他过程，但现在，只需要了解向服务器发出请求，然后服务器会使用浏览器用来向呈现信息的数据进行响应。

HTML

网站主要使用以下方式创建：

HTML，用于建立网站并定义其结构
CSS，通过添加样式选项使网站看起来更漂亮
JavaScript，使用交互功能在页面上实现复杂的功能

敏感数据泄露

当网站未能正确保护（或删除）最终用户的敏感明文信息时，就会发生敏感数据泄露；这些信息通常出现在网站的前端源代码中。

网站是使用许多 HTML 元素或标签构建的，只需“查看页面源代码”即可看到所有这些元素。网站开发人员可能忘记删除登录凭据、网站隐私部分的隐藏链接或 HTML 或 JavaScript 中显示的其他敏感数据。

敏感信息可能会被攻击者利用，进一步获取 Web 应用程序不同部分的访问权限。例如，可能存在带有临时登录凭据的 HTML 注释，如果查看了页面的源代码并发现了这一点，则可以使用这些凭据登录应用程序的其他地方（或者更糟的是，用于访问网站的其他后端组件）。

每当评估 Web 应用程序的安全性问题时，应该做的第一件事就是检查页面源代码，看看是否可以找到任何暴露的登录凭据或隐藏的链接。

HTML 注入

HTML 注入是一种漏洞，当页面上显示未经过滤的用户输入时就会发生这种情况。 如果网站未能过滤用户输入（过滤用户输入到网站的任何“恶意”文本），并且该输入在页面上使用，则攻击者可以将 HTML 代码注入易受攻击的网站。

输入清理对于确保网站安全非常重要，因为用户输入到网站的信息通常会用于其他前端和后端功能。将在另一个实验中探索的一个漏洞是数据库注入，可以通过控制查询中直接使用的输入来操纵数据库查找查询以作为另一个用户登录 - 但现在，专注于 HTML 注入（客户端）。

当用户可以控制其输入的显示方式时，他们可以提交 HTML（或 JavaScript）代码，浏览器将在页面上使用它，从而允许用户控制页面的外观和功能。
