Linux 的一些背景知识

Linux在哪里使用？

公平地说，Linux 比 Windows 等操作系统 (OS) 更难掌握。这两种操作系统各有优缺点。例如，Linux 轻量级得多，会惊讶地发现，很可能每天都以某种形式使用Linux ！ Linux支持以下功能：

访问的网站
汽车娱乐/控制面板
销售点 (PoS) 系统，例如商店的收银机和收银机
交通信号灯控制器或工业传感器等关键基础设施

Linux的特色

“Linux”这个名称实际上是基于 UNIX（另一种操作系统）的多个操作系统的总称。由于Linux是开源的，因此Linux的变体有各种形状和大小 - 最适合系统的用途。

例如，Ubuntu 和 Debian 是 Linux 的一些更常见的发行版，因为它具有很强的可扩展性。也就是说，可以将 Ubuntu 作为服务器（例如网站和 Web 应用程序）或功能齐全的桌面运行。在本系列中，将使用 Ubuntu。

Ubuntu Server 可以在只有 512MB RAM的系统上运行

与 Windows 有不同版本（7、8 和 10）类似，Linux也有许多不同的版本/发行版。

运行前几个命令

使用 Ubuntu 等操作系统的一大卖点是它非常轻便。当然，这也有其缺点，例如，通常没有 GUI（图形用户界面）或可以用来与机器交互的桌面环境（除非已安装）。与这些系统交互的很大一部分是使用“终端”。

“终端”是纯文本的，如果分解一些命令，

终端的样子如下
kali@linux1:~$ enter commands here
需要能够执行一些基本功能，例如导航到文件、输​​出其内容和创建文件！执行这些操作的命令是不言自明的（当然，一旦知道它是什么…）

让从下表中分解出来的两个命令开始：

命令 描述
echo 输出提供的任何文本
whoami 找出当前以哪个用户身份登录！

了解每个命令的使用示例

使用 echo
kali@linux1:~$ echo “Hello Friend!”

使用 whoami 找出登录的用户名
kali@linux1:~$ whoami

与文件系统交互！

当考虑需要做的事情时，它并不是很有用——包括浏览文件系统，以及对其进行读写。

将学习命令，以便能够做到这一点，将在下一个标题的表格中显示命令并展示使用这些命令的示例。

与文件系统交互

能够在不依赖桌面环境的情况下浏览登录的机器非常重要。毕竟，如果不能去任何地方，登录还有什么意义呢？

命令 描述
ls 列表
cd 更改目录
cat 连接
pwd 打印工作目录

列出当前目录中的文件（ls）

在执行任何操作（例如查找任何文件或文件夹的内容）之前，首先需要知道存在什么。这可以使用“ls”命令（listing 的缩写）来完成

使用“ls”列出当前目录的内容
kali@linux1:~$ ls
‘Important Files’ ‘My Documents’ Notes Pictures

在上面的截图中，可以看到有以下目录/文件夹：

重要文件
的文档
笔记
图片

根据文件夹名称，大概可以猜出文件夹里包含什么内容了。

可以使用ls和目录名称列出目录的内容，而无需导航到该目录。例如ls Pictures

改变当前目录 (cd)

现在知道了哪些文件夹存在，需要使用“ cd ”命令（change directory的缩写）切换到该目录。假设想打开“Pictures”目录 - 会执行“ cd Pictures ”。同样，想找出这个“Pictures”目录的内容，为此，再次使用“ ls ”：

使用“cd”后列出的新目录
kali@linux1:~/Pictures$ ls
dog_picture1.jpg dog_picture2.jpg dog_picture3.jpg dog_picture4.jpg
在这种情况下，看起来有 4 张狗的照片！

输出文件内容（cat）

虽然知道文件的存在很重要 - 但如果无法查看文件的内容，那么它就没什么用。

使用名为“ cat”的命令简单地查看文本文件的内容。

“Cat” 是 concatenating 的缩写，是输出文件内容（不仅仅是文本文件！）的绝佳方式。

如何结合使用“ls”来列出名为“Documents”的目录中的文件：

使用“ls”列出当前目录的内容
kali@linux1:/Documents$ ls
todo.txt
kali@linux1:/Documents$ cat todo.txt
Here’s something important for me to do later!

执行以下操作：

使用“ ls ”让知道这台机器的“Documents”文件夹中有哪些文件。在本例中，它被称为“todo.txt”。
然后，用来cat todo.txt连接/输出此“todo.txt”文件的内容，其中的内容是“这是稍后要做的一些重要的事情！”

可以使用cat 和目录名称cat来输出目录中文件的内容，而无需导航到该文件。例如cat /home/ubuntu/Documents/todo.txt

有时，用户名、密码、标志或配置设置等内容会存储在文件中，可以使用“cat”来检索这些内容。

找出当前工作目录的完整路径 (pwd)

当在Linux机器上导航时，当前正在工作的目录的名称将列在的终端中。

很容易忘记文件系统中到底在哪儿，“ pwd ” 代表打印工作目录。

使用之前的示例机器，目前位于“Documents”文件夹中 — 但这在 Linux 机器的文件系统上到底在哪里？可以使用此“pwd”命令找到它：

使用“pwd”列出当前目录的完整路径
kali@linux1:/Documents$ pwd
/home/ubuntu/Documents
kali@linux1:/Documents$
让详细分析一下：

通过终端，已经知道在“文档”中，但是此时，不知道“文档”存储在哪里，以便将来可以轻松地返回到它。
已经使用“ pwd ”（打印工作目录）命令来查找此“Documents”文件夹的完整文件路径。
Linux 帮助知道这个“Documents”目录存储在机器上的“/home/ubuntu/Documents”中
如果发现自己身处不同的位置，只需将cd /home/ubuntu/Documents的工作目录更改为这个“Documents”目录即可。

搜索文件

虽然目前看起来还不是这样，但 Linux 的优点之一就是它确实非常高效。

一个展示此类系统效率的绝妙方法是使用一组命令快速搜索整个系统中用户有权访问的文件。无需始终使用 cd 并 ls 找出文件的位置。相反，可以使用诸如此类的命令来 find 自动执行此类操作！

使用查找

find 命令非常棒，因为它既可以非常简单地使用，也可以相当复杂地使用，具体取决于要执行的具体操作。

以下面的代码片段为例，可以看到可用的目录列表：

使用“ls”列出当前目录的内容
kali@linux1:$ ls
Desktop Documents Pictures folder1
kali@linux1:$

桌面
文件
图片
文件夹1
当然，现在目录本身可以包含更多目录。当必须逐个查看目录以查找特定文件时，这就会变得很麻烦。可以使用find来为做到这一点！

假设已经知道要找的文件的名字——但不记得它到底在哪里！在本例中，正在寻找“passwords.txt”

如果记得文件名，可以简单地使用find -name passwords.txt命令来查找当前目录中的每个文件夹以查找该特定文件，如下所示：

使用“find”查找名为“passwords.txt”的文件
kali@linux1:$ find -name passwords.txt
./folder1/passwords.txt
kali@linux1:$

“查找”成功找到了文件 — 原来它位于 folder1/passwords.txt — 太棒了。但是，假设不知道文件的名称，或者想要搜索每个带有“.txt”等扩展名的文件。查找让也能做到这一点！

可以简单地使用通配符 (*) 来搜索以 .txt 结尾的任何内容。在的例子中，想要查找当前目录中的每个 .txt 文件。将构造一个命令，例如find -name *.txt。其中“Find”能够找到每个 .txt 文件，然后为提供了每个文件的位置：

使用“find”查找任何扩展名为“.txt”的文件
kali@linux1:$ find -name *.txt
./folder1/passwords.txt
./Documents/todo.txt
kali@linux1:$

查找已成功找到：

“passwords.txt”位于“folder1”内
“todo.txt”位于“文档”中

使用 Grep

grep命令允许在文件内容中搜索正在寻找的特定值。

以 Web 服务器的访问日志为例，本例中 Web 服务器的 access.log 有 244 条记录。

使用“wc”计算“access.log”中的条目数
kali@linux1:$ wc -l access.log
244 access.log
kali@linux1:$

使用类似这样的命令cat在这里效果不太好。例如，如果想搜索此日志文件以查看某个用户/IP 地址访问的内容？考虑到想要找到一个特定的值，查看 244 个条目并不是那么有效。

可以使用grep 搜索此文件的整个内容来查找要搜索的值的任何条目。以 Web 服务器的访问日志为例，希望查看 IP 地址“81.143.211.90”访问过的所有内容

使用“grep”在“access.log”中查找 IP 地址为“81.143.211.90”的任何条目
kali@linux1:$ grep “81.143.211.90” access.log
81.143.211.90 - - [25/Mar/2021:11:17 + 0000] “GET / HTTP/1.1” 200 417 “-“ “Mozilla/5.0 (Linux; Android 7.0; Moto G(4))”
kali@linux1:$
“Grep” 搜索了此文件并向展示了提供的所有条目以及包含在 IP 的此日志文件中的内容。

Shell 运算符简介

Linux 运算符是增强使用 Linux 知识的绝佳方式。有几个重要的运算符值得注意。

符号/运算符 描述
& 该操作符允许在终端后台运行命令。
&& 该运算符允许在终端的一行中将多个命令组合在一起。
> 该操作符是一个重定向器 - 这意味着可以获取命令的输出（例如使用 cat 来输出文件）并将其指向其他地方。
>> 该运算符执行与运算符 > 相同的功能，但附加输出而不是替换（意味着没有覆盖任何内容）。

运算符 ”＆”

此运算符允许在后台执行命令。例如，假设要复制一个大文件。这显然会花费相当长的时间，并且将无法执行任何其他操作，直到文件成功复制为止。

“&” shell 运算符允许执行一个命令并让它在后台运行（例如此文件复制），从而让可以做其他事情

运算符 ”＆＆”

这个 shell 运算符有点误导人，因为它与它的搭档“&”很相似。与“&”运算符不同，可以使用“&&”列出要运行的命令，例如command1 && command2。但是，值得注意的是，只有成功command2时才会运行。command1

运算符 “>”

这个操作符就是所谓的输出重定向器。这本质上意味着从运行的命令中获取输出并将该输出发送到其他地方。

运行echo howdy会将“howdy”返回到的终端 — — 这不是很有用。可以做的是将“howdy”重定向到某个东西，例如一个新文件

假设想创建一个名为“welcome”的文件，内容为“hey”。可以echo hey > welcome在想要创建文件的地方运行，内容为“hey”，如下所示：

使用 > 运算符
kali@linux1:$ echo hey > welcome
使用 cat 输出“welcome”文件
kali@linux1:$ cat welcome
hey

如果文件即“welcome”已经存在，其内容将被覆盖！

运算符“>>”

与上一个运算符 ( >) 类似，此运算符也是一个输出重定向器。但是，此运算符的不同之处在于，它不会覆盖文件中的任何内容，而是将输出放在末尾。

继续之前的例子，有一个文件“welcome”，其中包含“hey”的内容。如果使用 echo 使用运算符将​​“hello”添加到文件> ，则文件现在将只有“hello”而没有“hey”。

该>>操作符允许将输出附加到文件底部 — 而不是像这样替换内容：

使用 >> 运算符
kali@linux1:$ echo hello >> welcome
使用 cat 输出“welcome”文件
kali@linux1:$ cat welcome
hey
hello
