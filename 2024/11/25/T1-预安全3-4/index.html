综合起来

当在浏览器中请求网页时，幕后会发生很多事情。当请求网站时，的计算机需要知道它需要与之通信的服务器的 IP 地址；为此，它使用DNS。然后，的计算机使用一组称为HTTP协议的特殊命令与 Web 服务器通信；然后，Web 服务器返回 HTML、JavaScript、CSS、图像等，然后的浏览器将使用它们正确格式化并向显示网站。

还有一些其他组件可以帮助网络更高效地运行并提供额外的功能。

其他组件

负载均衡器

当网站的流量开始变得非常大或正在运行需要高可用性的应用程序时，一台 Web 服务器可能不再能胜任。负载平衡器提供两项主要功能，确保高流量网站能够处理负载，并在服务器无响应时提供故障转移。
当使用负载均衡器请求网站时，负载均衡器将首先接收的请求，然后将其转发到其背后的多台服务器之一。负载均衡器使用不同的算法来帮助它决定哪台服务器最适合处理请求。这些算法的几个示例是 循环算法，它将请求依次发送到每台服务器，或 加权算法，它会检查服务器当前正在处理多少个请求，并将其发送到最不繁忙的服务器。

负载均衡器还会定期检查每台服务器，以确保它们正常运行；这称为 健康检查。如果服务器没有正确响应或没有响应，负载均衡器将停止发送流量，直到它再次正确响应。

CDN（内容分发网络）

CDN 是减少繁忙网站流量的绝佳资源。它允许托管网站上的静态文件（例如 JavaScript、CSS、图像、视频），并将它们托管在世界各地的数千台服务器上。当用户请求其中一个托管文件时，CDN 会找出最近的服务器的物理位置，并将请求发送到那里，而不是可能发送到世界的另一端。

数据库

网站通常需要一种为用户存储信息的方式。Web 服务器可以与数据库通信，以存储和调用其中的数据。数据库可以是简单的纯文本文件，也可以是提供速度和弹性的复杂多服务器集群。会遇到一些常见的数据库：MySQL、MSSQL、MongoDB、Postgres 等；每个数据库都有其特定的功能。

WAF（Web 应用程序防火墙）

WAF 位于的 Web 请求和 Web 服务器之间；其主要目的是保护 Web 服务器免受黑客攻击或拒绝服务攻击。它会分析 Web 请求中常见的攻击技术，无论请求是来自真实浏览器还是机器人。它还会利用所谓的速率限制来检查是否发送了过多的 Web 请求，速率限制将仅允许每秒从 IP 发出一定数量的请求。如果请求被视为潜在攻击，它将被丢弃并且永远不会发送到 Web 服务器。


Web 服务器的工作原理

Web 服务器如何工作。

什么是 Web 服务器？

Web 服务器是一种软件，它监听传入的连接，然后利用 HTTP 协议向其客户端提供 Web 内容。遇到的最常见的 Web 服务器软件是 Apache、Nginx、IIS 和 NodeJS。Web 服务器从其根目录提供文件，该目录在软件设置中定义。例如，Nginx 和 Apache 在 Linux 操作系统中共享相同的默认位置 /var/www/html，而 IIS 在 Windows 操作系统中使用 C:\inetpub\wwwroot。因此，例如，如果请求文件 http://www.example.com/picture.jpg，它将从其本地硬盘发送文件 /var/www/html/picture.jpg。

虚拟主机

Web 服务器可以托管具有不同域名的多个网站；为了实现这一点，它们使用虚拟主机。Web 服务器软件从HTTP标头检查所请求的主机名，并将其与其虚拟主机进行匹配（虚拟主机只是基于文本的配置文件）。如果找到匹配项，则会提供正确的网站。如果未找到匹配项，则会提供默认网站。

虚拟主机可以将其根目录映射到硬盘上的不同位置。例如，one.com映射到 /var/www/website_one，two.com映射到 /var/www/ website_two

可以在网络服务器上托管的不同网站的数量没有限制。

静态内容与动态内容

静态内容，顾名思义，是永远不会改变的内容。常见的例子有图片、javascript、CSS 等，但也可以包括永远不会改变的 HTML。此外，这些文件直接从网络服务器提供，不会对其进行任何更改。

另一方面，动态内容是可以随着不同请求而变化的内容。以博客为例。在博客的主页上，它会显示最新条目。如果创建了新条目，则主页会更新为最新条目，或者第二个示例可能是博客上的搜索页面。根据搜索的单词，将显示不同的结果。

最终看到的这些变化都是在所谓的 后端 使用编程和脚本语言完成的。之所以称为后端，是因为所做的一切都是在幕后完成的。无法查看网站的 HTML 源代码并查看后端发生了什么，而 HTML 是后端处理的结果。在浏览器中看到的所有内容都称为前端 。

脚本和后端语言

后端语言所能实现的功能没有太大限制，这些功能使网站与用户具有交互性。这些语言的一些示例（无特定顺序 :p）包括 PHP、Python、Ruby、NodeJS、Perl 等等。这些语言可以与数据库交互、调用外部服务、处理来自用户的数据等等。一个非常基本的 PHP 示例是，如果请求网站 http://example.com/index.php?name=adam

如果 index.php 是这样构建的：

Hello

它将向客户端输出以下内容：

Hello adam

会注意到，客户端看不到任何 PHP 代码，因为它位于 Backend上。正如在后续模块中了解到的那样，这种交互性为尚未安全创建的 Web 应用程序带来了更多安全问题。
