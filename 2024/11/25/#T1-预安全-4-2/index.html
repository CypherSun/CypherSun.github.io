使用 SSH 访问的 Linux 机器（部署）

浏览器内的功能使用的协议与今天要使用的协议完全相同。该协议称为S ecure Shell或简称SSH ，是连接远程Linux计算机的命令行并与之交互的常用方法。

什么是SSH以及它如何工作？

安全 Shell 或SSH只是一种加密形式的设备间协议。使用加密技术，以人类可读格式发送的任何输入都会被加密以在网络上传输 - 而一旦到达远程计算机，它就会被解密，如下图所示。

需要了解：

SSH允许远程在另一台设备上执行命令。
设备之间发送的任何数据在通过互联网等网络发送时都会被加密

部署Linux机器

使用 SSH 登录到的Linux机器

使用SSH的语法非常简单。只需要提供两件事：

  1. 远程机器的 IP 地址

  2. 将凭据更正为有效帐户，以便在远程计算机上登录

例如：ssh cyphersun@MACHINE_IP 。将 IP 地址替换为的 Linux 目标机器的 IP 地址。执行后，将被要求信任主机。

现在已经连接，执行的任何命令都将在远程机器上执行 - 而不是自己的机器上。

注意：当在 ssh 登录提示中输入密码时，没有任何可见的反馈 - 在输入密码时看不到任何文本或符号。但它仍然有效，因此只需输入密码并按 Enter 即可登录。

标志和开关简介

大多数命令允许提供参数。这些参数由连字符和特定关键字（称为标志或开关）标识。

稍后将讨论如何识别哪些命令允许提供参数以及理解这些命令的具体作用。

使用命令时，除非另有说明，否则它将执行其默认行为。例如， ls 列出工作目录的内容。但是，不会显示隐藏文件。可以使用标志和开关来扩展命令的行为。

使用的ls示例，ls告诉只有一个名为“folder1”的文件夹，如下面的屏幕截图所示。 请注意，以下屏幕截图中的内容仅为示例。

使用 ls 查看目录内容
tryhackme@linux2:$ ls
folder1
tryhackme@linux2:$
然而，在使用-a参数（ 的缩写–all）后，现在突然得到了一个输出，其中包含更多文件和文件夹，例如“.hiddenfolder”。带有“ . ”的文件和文件夹是隐藏文件。

使用 ls 查看隐藏文件夹
tryhackme@linux2:$ ls -a
.hiddenfolder folder1
tryhackme@linux2:$

创建文件和文件夹（touch、mkdir）

在 Linux 上创建文件和文件夹是一个简单的过程。首先，将介绍如何创建文件。touch 命令只接受一个参数——想要为创建的文件指定的名称。例如，可以使用 创建文件“note” touch note。值得注意的是，touch 只会创建一个空白文件。需要使用 echo 等命令或 nano 等文本编辑器将内容添加到空白文件中。

使用触摸创建新文件
tryhackme@linux2:$ touch note
tryhackme@linux2:$ ls
folder1 note
这与创建文件夹的过程类似，只需使用mkdir命令并再次提供要分配给目录的名称即可。例如，使用 创建目录“mydirectory” mkdir mydirectory。

使用 mkdir 创建新目录
tryhackme@linux2:$ mkdir mydirectory
tryhackme@linux2:$ ls
folder1 mydirectory note

删除文件和文件夹（rm）

rm是迄今为止介绍过的命令中最特别的。可以使用 轻松删除文件rm。但是，需要-R在要删除的目录名称旁边提供开关。

使用 rm 删除文件
tryhackme@linux2:$ rm note
tryhackme@linux2:$ ls
folder1 mydirectory
递归使用 rm 删除目录
tryhackme@linux2:$ rm -R mydirectory
tryhackme@linux2:$ ls
folder1

复制和移动文件和文件夹（cp、mv）

复制和移动文件是Linux机器上的一项重要功能。从 开始cp，此命令接受两个参数：

  1. 现有文件的名称

  2. 复制时希望分配给新文件的名称

cp将现有文件的全部内容复制到新文件中。在下面的屏幕截图中，将“note”复制到“note2”。

使用 cp 复制文件
tryhackme@linux2:$ cp note note2
tryhackme@linux2:$ ls
folder1 note note2
移动文件需要两个参数，就像 cp 命令一样。但是，与复制和/或创建新文件不同，mv它将合并或修改作为参数提供的第二个文件。不仅可以使用它mv来将文件移动到新文件夹，还可以使用它mv 来重命名文件或文件夹。例如，在下面的屏幕截图中，将文件“note2”重命名为“note3”。“note3”现在将包含“note2”的内容。

使用 mv 移动文件
tryhackme@linux2:$ mv note2 note3
tryhackme@linux2:$ ls
folder1 note note3

确定文件类型

经常会误导人并使他措手不及的是从文件中推测其用途或内容。文件通常具有所谓的扩展名，以使此操作更容易。例如，文本文件的扩展名通常为“.txt”。但这不是必需的。

到目前为止，在示例中使用的文件还没有扩展名。如果不知道文件存在的原因，就无法真正了解它的用途。输入命令file。此命令接受一个参数。例如，将使用file来确认示例中的“note”文件是否确实是文本文件，如下所示file note。

使用文件确定文件的内容
tryhackme@linux2:~$ file note
note: ASCII text

权限 101

现在可能已经发现，某些用户无法访问某些文件或文件夹。之前探索过一些命令，这些命令可用于确定拥有哪些访问权限以及访问权限将引导到哪里。

在之前的任务中，学习了如何通过标志和开关扩展命令的使用。以列出ls当前目录内容的命令为例。使用-l 开关时，可以看到十列，如下面的屏幕截图所示。但是，只对前三列感兴趣：

使用 ls -lh 列出目录中所有文件的权限
tryhackme@linux2:~$ ls -lh
-rw-r–r– 1 cmnatic cmnatic 0 Feb 19 10:37 file1

虽然令人生畏，但这三列对于确定文件或文件夹的某些特征以及是否有权访问它非常重要。文件或文件夹可以有几个特征，这些特征决定了允许哪些操作以及哪个用户或组有能力执行给定的操作——例如以下内容：

让使用本任务顶部的初始屏幕截图中的“cmnatic.pem”文件。它有“-”指示符，突出显示它是一个文件，后面跟着“rw”。这意味着只有文件的所有者可以读取和写入此“cmnatic.pem”文件，但不能执行它。

用户和组之间的区别

Linux的优点在于权限可以非常细化，虽然用户在技术上拥有一个文件，但如果设置了权限，那么一组用户也可以对完全相同的文件拥有相同或不同的权限集，而不会影响文件所有者本身。

让将其放到现实世界中；运行 Web 服务器的系统用户必须具有读写文件的权限，才能使 Web 应用程序有效运行。但是，诸如网络托管公司之类的公司必须允许其客户在不成为 Web 服务器系统用户的情况下将自己的文件上传到其网站 - 这会危及所有其他客户的安全。

将学习在下面的用户之间切换所需的命令。

在用户之间切换

借助该命令，在Linux安装中切换用户变得非常简单su。除非是 root 用户（或通过 sudo 使用 root 权限），否则需要了解两件事才能实现用户帐户的转换：

希望切换到的用户
用户密码
该su命令需要几个可能与相关的开关。例如，登录后执行命令或指定要使用的特定 shell。 鼓励阅读手册页以su了解更多信息。不过，将介绍-l或–login开关。

简单地说，通过提供-l 开关su，启动一个与登录系统的实际用户更相似的 shell - 继承了新用户的更多属性，即环境变量等。

使用 su 交互切换到用户 2
tryhackme@linux2:~$ su user2
Password:
user2@linux2:/home/tryhackme$
例如，当使用su切换到“user2”时，的新会话会将带入前一个用户的主目录。

使用 su 交互切换到用户 2
tryhackme@linux2:$ su -l user2
Password:
user2@linux2:$ pwd
user2@:/home/user2$
现在，使用之后-l，的新会话已自动将带入“用户”的主目录。

公用目录

/ETC

此根目录是系统上最重要的根目录之一。etc 文件夹（etcetera 的缩写）是存储操作系统使用的系统文件的常用位置。

例如，下面屏幕截图中突出显示的 sudoers 文件包含有权以 root 用户身份运行 sudo 或一组命令的用户和组列表。

下面还重点介绍了“ passwd ”和“ shadow ”文件。这两个文件是Linux特有的，因为它显示了系统如何以称为 sha512 的加密格式存储每个用户的密码。

/etc 目录中一些值得注意的内容
tryhackme@linux2:/etc$ ls
shadow passwd sudoers sudoers.d

/var

“/var”目录（“var”是变量数据的缩写），是Linux安装中的主要根文件夹之一。此文件夹存储系统上运行的服务或应用程序经常访问或写入的数据。例如，正在运行的服务和应用程序的日志文件就写在这里（/var/log），或者其他不一定与特定用户相关的数据（即数据库等）。

/var 目录中一些值得注意的内容
tryhackme@linux2:/var$ ls
backups log opt tmp

/root

与/home目录不同，/root 文件夹实际上是“root”系统用户的家。除了了解这是“root”用户的家目录之外，此文件夹没有其他内容。但是，值得一提的是，逻辑上可以假设该用户默认将其数据放在“ / home/root ”之类的目录中。

/root 目录中一些值得注意的内容
root@linux2:~# ls
myfile myfolder passwords.xlsx

/tmp

这是 Linux 安装中一个独特的根目录。/tmp 目录是“临时”的缩写，它是易失性的，用于存储只需要访问一两次的数据。与计算机上的内存类似，一旦计算机重新启动，此文件夹的内容就会被清除。

对于进行渗透测试来说，任何用户默认都可以写入此文件夹，这很有用。这意味着，一旦能够访问某台机器，它就可以成为存储枚举脚本等内容的好地方。

/tmp 目录中一些值得注意的内容
root@linux2:/tmp# ls
todelete trash.txt rubbish.bin
