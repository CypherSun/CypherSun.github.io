搜索结果评估

对各种文章/文件/杂志/信息来源的评价

在互联网上，每个人都可以发表自己的文章。它可以是博客文章、文章或社交媒体帖子的形式。它甚至可以以更微妙的方式发表，例如通过编辑公共维基页面。这种能力使任何人都可以表达他们毫无根据的主张。每个人都可以表达他们对最佳网络安全实践、未来编程趋势以及如何最好地准备DevSecOps面试的看法。

作为读者，评估信息是我们的职责。我们将提到评估信息时需要考虑的几点：

来源：确定发布信息的作者或组织。考虑他们是否在该主题上享有声誉和权威。发表博客文章并不代表某人在该主题上具有权威性。
证据和推理：检查主张是否有可靠的证据和逻辑推理。我们寻求确凿的事实和有力的论据。
客观性和偏见性：评估信息是否公正合理地呈现，是否反映了多种观点。我们不关心作者推动阴暗议程，无论是为了推销产品还是攻击竞争对手。
确证和一致性：通过多个独立来源的确证来验证所呈现的信息。检查多个可靠且信誉良好的来源是否同意核心主张。

搜索引擎

我们每个人都使用过互联网搜索引擎；然而，并不是每个人都尝试过充分利用互联网搜索引擎的全部功能。几乎每个互联网搜索引擎都允许您进行高级搜索。请考虑以下示例：

让我们考虑一下 Google 支持的搜索运算符。

“exact phrase”：双引号表示您正在查找包含精确单词或短语的页面。例如，可以搜索”passive reconnaissance”以获取包含此精确短语的页面。
site:：此运算符允许您指定要限制搜索的域名。例如，我们可以使用 搜索 TryHackMe 上的成功案例site:tryhackme.com success stories。
-：减号允许您忽略包含特定单词或短语的搜索结果。例如，您可能有兴趣了解金字塔，但不想查看旅游网站；一种方法是搜索pyramids -tourism或-tourism pyramids。
filetype:：此搜索运算符对于查找文件（而不是网页）必不可少。您可以使用 Google 搜索的一些文件类型包括便携式文档格式 (PDF)、Microsoft Word 文档 (DOC)、Microsoft Excel 电子表格 (XLS) 和 Microsoft PowerPoint 演示文稿 (PPT)。例如，要查找网络安全演示文稿，请尝试搜索filetype:ppt cyber security。
您可以在此高级搜索运算符列表中查看各种搜索引擎中的更多高级控件；但是，以上内容提供了一个很好的起点。查看您最喜欢的搜索引擎支持的搜索运算符。

专业搜索引擎

您熟悉互联网搜索引擎；但是，您对专业搜索引擎有多少了解呢？我们所说的专业搜索引擎是指用于查找特定类型结果的搜索引擎。

初段
让我们从Shodan开始，这是一个用于搜索连接到互联网的设备的网站。它允许您搜索特定类型和版本的服务器、网络设备、工业控制系统和物联网设备。您可能想看看有多少服务器仍在运行Apache 2.4.1，以及它在各个国家的分布情况。为了找到答案，我们可以搜索apache 2.4.1，它将返回标头中包含字符串“apache 2.4.1”的服务器列表。

在 Shodan 网站上搜索 apache 2.4.1 的结果。

可以考虑访问 Shodan搜索查询示例以获取更多示例。此外，如果您有订阅，还可以查看Shodan 趋势以获取历史见解。

森西斯
乍一看，Censys似乎与 Shodan 相似。但是，Shodan 专注于与互联网连接的设备和系统，例如服务器、路由器、网络摄像头和物联网设备。另一方面，Censys 专注于与互联网连接的主机、网站、证书和其他互联网资产。它的一些用例包括枚举正在使用的域、审计开放端口和服务以及发现网络中的恶意资产。您可能需要查看Censys 搜索用例。

在 Censys 网站上搜索 apache 2.4.1 的结果。

我被黑了吗
Have I Been Pwned (HIBP) 的作用是告诉您电子邮件地址是否出现在泄露的数据泄露中。在泄露的数据中找到某人的电子邮件表明泄露了私人信息，更重要的是密码。许多用户在多个平台上使用相同的密码，如果一个平台被泄露，他们在其他平台上的密码也会被泄露。事实上，密码通常以加密格式存储；然而，许多密码并不那么复杂，可以使用各种攻击来恢复。


漏洞和利用

漏洞漏洞
我们可以将通用漏洞披露 ( CVE ) 计划视为漏洞词典。它为软件和硬件产品中的漏洞和安全问题提供了标准化标识符。每个漏洞都分配有一个CVE ID，其标准化格式如下CVE-2024-29988。此唯一标识符 (CVE ID) 可确保从安全研究人员到供应商和 IT 专业人员的每个人都指的是同一个漏洞，在本例中为CVE -2024-29988。

MITRE 公司负责维护 CVE 系统。如需更多信息和搜索现有 CVE，请访问CVE计划网站。或者，访问国家漏洞数据库(NVD) 网站。以下屏幕截图显示了CVE -2014-0160，也称为 Heartbleed。

CVE 计划网站上的 CVE-2014-0160 漏洞

漏洞数据库
想要利用存在漏洞的应用程序的原因有很多；其中一个原因是作为红队的一员评估公司的安全性。毋庸置疑，除非获得许可（通常是通过具有法律约束力的协议），否则我们不应该试图利用存在漏洞的系统。

既然我们有权利用易受攻击的系统，我们可能需要找到一个有效的漏洞代码。一个资源是漏洞数据库。漏洞数据库列出了来自不同作者的漏洞代码；其中一些漏洞代码已经过测试并标记为已验证。

技术文档

需要掌握的一项重要技能是查阅官方文档。我们将介绍一些官方文档页面的示例。

Linux手册页
早在互联网普及之前，您如何在Linux或类 Unix 系统中获取使用命令的帮助？答案是查看手册页（简称 man page）。在Linux和每个类 Unix 系统上，每个命令都应该有一个手册页。事实上，系统调用、库函数甚至配置文件也存在手册页。

假设我们要查看命令的手册页ip。我们发出命令man ip。下面的屏幕截图显示了我们收到的页面。您可能想要启动 AttackBox 并man ip在终端上运行。按q退出。

ip 命令的手册页

ip如果您希望在 Web 浏览器中阅读手册页，只需man ip在您最喜欢的搜索引擎中输入即可。此页面可能位于结果顶部。

AttackBox是一个可通过浏览器访问的Linux系统。单击“启动 AttackBox”按钮将在分屏中显示 AttackBox，方便在同一浏览器窗口中阅读任务文本和应用说明。如果您隐藏了 AttackBox 窗口，则可以通过单击顶部的蓝色“显示分屏视图”按钮再次显示它。在此任务中，您可以启动 AttackBox 并使用它来尝试Linux命令，例如。man

产品文档
每一个流行的产品都应该有组织良好的文档。这些文档提供了有关产品特性和功能的官方和可靠的信息来源。示例包括Snort 官方文档、Apache HTTP服务器文档、PHP文档和Node.js 文档。

查看官方文档总是有益的，因为它是最新的并提供最完整的产品信息。

社交媒体

有笔记本电脑的黑客

社交媒体平台上注册的用户数以亿计，例如Facebook、Twitter和LinkedIn。我们希望您熟悉热门平台。但是，如果您知道任何不熟悉的平台，我们建议您查看并了解它。理想情况下，人们希望在不创建账户的情况下探索平台；然而，这会严重限制您的体验。相反，我们建议您使用临时电子邮件地址来发现这些平台，而无需将它们链接到您的真实电子邮件地址；完成后，您可以终止帐户和关联的电子邮件地址。不使用主帐户的一个原因是，您不希望在您临时探索平台时，您的联系人开始在那里与您联系。

社交媒体的强大之处在于它能让你与你感兴趣的公司和人建立联系。此外，社交媒体为网络安全专业人员提供了丰富的信息，无论他们是在搜索人员还是技术信息。你可能会问，为什么搜索人员很重要？

在保护公司时，你应该确保你保护的人不会在社交媒体上过度分享信息。例如，他们的社交媒体可能会泄露他们秘密问题的答案，例如“你小时候上哪所学校？”。这些信息可能会让对手轻松重置他们的密码并接管他们的账户。

此外，作为网络安全专业人员，您需要随时了解新的网络安全趋势、技术和产品。关注适当的渠道和团体可以为您提供发展技术专长的合适环境。

除了通过社交媒体渠道和群组了解最新动态外，我们还应该提到新闻媒体。数百个新闻网站会提供有价值的网络安全相关新闻。尝试不同的新闻网站，并坚持使用您最喜欢的新闻网站。
