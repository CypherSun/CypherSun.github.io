什么是HTTP？（超文本传输​​协议）

HTTP是浏览网站时使用的协议，由 Tim Berners-Lee 及其团队在 1989 年至 1991 年间开发。HTTP是用于与 Web 服务器通信以传输网页数据的一组规则，无论是 HTML、图像、视频等。

什么是 HTTPS？（超文本传输​​协议安全）

HTTPS 是 HTTP 的安全版本。HTTPS 数据经过加密，因此不仅可以阻止他人查看接收和发送的数据，还可以确保正在与正确的 Web 服务器通信，而不是与冒充它的服务器通信。

请求和响应

当访问网站时，浏览器需要向 Web 服务器请求 HTML、图片等资源，并下载响应。在此之前，需要明确告诉浏览器如何以及在何处访问这些资源，这时 URL 就可以派上用场了。

什么是 URL？（统一资源定位器）

如果使用过互联网，那么以前就使用过 URL。URL 主要是有关如何访问互联网上的资源的说明。下图显示了 URL 及其所有功能的样子（它不会在每个请求中使用所有功能）。

示例上显示 URL 不同部分的图表，其中 http 是方案，user:password 是用户，cyphersun.com 是域或主机，80 是端口，view-room 是路径，?id=1 是查询字符串，#task3 是片段。完整地址为 http://user:password@cyphersun.com:80/view-room?id=1#task3。

方案：这指示使用什么协议来访问资源，例如 HTTP、HTTPS、FTP（文件传输协议）。

用户：有些服务需要认证才能登录，可以将用户名和密码放入URL中登录。

主机：要访问的服务器的域名或 IP 地址。

端口：要连接的端口，通常HTTP为 80 ，HTTPS 为 443，但可以托管在 1 - 65535 之间的任何端口上。

路径：尝试访问的资源的文件名或位置。

查询字符串：可以发送到请求路径的额外信息。例如，/blog? id=1会告诉博客路径希望接收 id 为 1 的博客文章。

片段：这是对实际请求的页面上某个位置的引用。这通常用于内容较长的页面，并且可以将页面的某个部分直接链接到它，这样用户一访问该页面就可以立即看到它。

提出请求

只需一行“ GET / HTTP /1.1 ”即可向 Web 服务器发出请求

但为了获得更丰富的网络体验，还需要发送其他数据。这些其他数据以所谓的标头形式发送，标头包含要提供给正在通信的网络服务器的额外信息，但将在标头任务中对此进行更深入的介绍。

示例请求：

GET / HTTP/1.1
Host: cyphersun.com
User-Agent: Mozilla/5.0 Firefox/87.0
Referer: https://cyphersun.com/

细分此请求的每一行：

第 1 行：此请求发送 GET 方法（有关更多信息，请参阅HTTP方法任务），使用 / 请求主页，并告知 Web 服务器正在使用HTTP协议版本 1.1。

第 2 行：告诉 Web 服务器想要访问网站 cyphersun.com

第 3 行：告诉 Web 服务器正在使用 Firefox 87 版浏览器

第 4 行：告诉 Web 服务器，引导访问此网站的网页是https://cyphersun.com

第 5 行：HTTP请求总是以一个空白行结束，以通知 Web 服务器请求已完成。

响应示例：

HTTP/1.1 200 OK
Server: nginx/1.15.8
Date: Fri, 09 Apr 2021 13:34:03 GMT
Content-Type: text/html
Content-Length: 98

cyphersun Welcome To cyphersun.com 分解响应的每一行：

第 1 行： HTTP 1.1 是服务器正在使用的 HTTP 协议的版本，后面跟着 HTTP 状态代码（在本例中为“200 Ok”），这告诉请求已成功完成。

第 2 行：这告诉 Web 服务器软件和版本号。

第 3 行： Web 服务器的当前日期、时间和时区。

第 4 行： Content-Type 标头告诉客户端要发送什么类型的信息，例如 HTML、图像、视频、pdf、XML。

第 5 行： Content-Length 告诉客户端响应有多长，这样就可以确认没有丢失任何数据。

第 6 行： HTTP响应包含一个空白行，以确认HTTP响应的结束。

第 7-14 行：已请求的信息，在本例中为主页。

HTTP 方法

HTTP 方法是客户端在发出 HTTP 请求时显示其预期操作的一种方式。HTTP 方法有很多，但将介绍最常见的方法，尽管主要会处理 GET 和 POST 方法。

GET请求

这用于从网络服务器获取信息。

POST请求

这用于向 Web 服务器提交数据并可能创建新记录

PUT请求

用于向网络服务器提交数据以更新信息

Delete请求

这用于从网络服务器删除信息/记录。

HTTP 状态代码

HTTP 状态代码：

在上一个任务中，了解到，当 HTTP 服务器响应时，第一行始终包含一个状态代码，用于通知客户端其请求的结果以及可能的处理方式。这些状态代码可分为 5 个不同的范围：

100-199 - 信息响应 这些代码用于告知客户端其请求的第一部分已被接受，客户端应继续发送其请求的其余部分。这些代码不再很常见。
200-299 - 成功 这一系列的状态代码用于告知客户端他们的请求已成功。
300-399 - 重定向 这些用于将客户端的请求重定向到另一个资源。这可以是不同的网页，也可以是完全不同的网站。
400-499 - 客户端错误 用于通知客户他们的请求有错误。
500-599 - 服务器错误 这是为服务器端发生的错误而保留的，通常表示服务器处理请求时存在相当大的问题。

常见的HTTP状态代码：

有很多不同的 HTTP 状态代码，而且应用程序甚至可以定义自己的状态代码，将介绍可能遇到的最常见的HTTP响应：

200——确定 请求已成功完成。
201——已创建 已创建资源（例如新用户或新博客文章）。
301 - 永久移动 这会将客户端的浏览器重定向到新的网页或告诉搜索引擎该页面已移动到其他地方并改为在那里查看。
302 - 已找到 与上面的永久重定向类似，但顾名思义，这只是暂时的改变，在不久的将来可能会再次改变。
400 - 错误请求 这告诉浏览器他们的请求中存在错误或缺失。如果正在请求的 Web 服务器资源需要客户端未发送的某个参数，则有时可以使用此功能。
401-未授权 目前，不被允许查看此资源，除非获得 Web 应用程序的授权（通常是用户名和密码）。
403 - 禁止访问 无论是否登录，都没有权限查看此资源。
405 - 方法不允许 资源不允许此方法请求，例如，向资源 /create-account 发送了一个 GET 请求，而它原本期望一个 POST 请求。
404 - 页面未找到 请求的页面/资源不存在。
500 - 内部服务错误 服务器在处理的请求时遇到某种错误，不知道如何正确处理。
503 - 服务不可用
该服务器无法处理的请求，因为它超载或正在维护。


标头

标头是在发出请求时可以发送到 Web 服务器的附加数据位。

尽管在发出HTTP请求时没有严格要求标头，但会发现很难正确查看网站。

通用请求标头

这些是从客户端（通常是的浏览器）发送到服务器的标头。

主机：一些网络服务器托管多个网站，因此通过提供主机标头，可以告诉它需要哪一个网站，否则只会收到服务器的默认网站。

用户代理：这是的浏览器软件和版本号，告诉网络服务器的浏览器软件有助于它为的浏览器正确格式化网站，并且 HTML、JavaScript 和 CSS 的某些元素仅在某些浏览器中可用。

Content-Length：当以表单等形式向 Web 服务器发送数据时，内容长度会告知 Web 服务器在 Web 请求中需要多少数据。这样，服务器就可以确保不会丢失任何数据。

Accept-Encoding：告诉网络服务器浏览器支持哪些类型的压缩方法，以便数据可以变得更小，以便在互联网上传输。

Cookie：发送到服务器以帮助记住的信息的数据（有关更多信息，请参阅 cookies 任务）。
常见响应标头

这些是请求后从服务器返回给客户端的标头。

Set-Cookie：存储每次请求时发送回 Web 服务器的信息（有关更多信息，请参阅 cookies 任务）。

Cache-Control：在浏览器再次请求之前，将响应内容存储在浏览器缓存中多长时间。

Content-Type：这告诉客户端返回的数据类型，即 HTML、CSS、JavaScript、图像、PDF、视频等。使用 content-type 标头，浏览器就知道如何处理数据。

内容编码：在互联网上发送数据时，使用了什么方法压缩数据以使其更小。

曲奇饼

Cookie 只是存储在计算机上的一小段数据。当从 Web 服务器收到“Set-Cookie”标头时，Cookie 就会保存。然后，发出的每一个后续请求都会将 Cookie 数据发送回 Web 服务器。由于 HTTP 是无状态的（不会跟踪之前的请求），因此 Cookie 可用于提醒 Web 服务器是谁、网站的一些个人设置或是否曾经访问过该网站。以HTTP请求为例来看一下：

Cookie 有多种用途，但最常用于网站身份验证。Cookie 值通常不是可以看到密码的明文字符串，而是一个令牌（人类无法轻易猜出的唯一密码）。

查看的 Cookie

可以使用浏览器中的开发人员工具轻松查看浏览器向网站发送的 cookie。如果不确定如何在浏览器中访问开发人员工具，请单击此任务顶部的“查看网站”按钮以获取操作指南。

打开开发者工具后，点击“网络”选项卡。此选项卡将显示浏览器请求的所有资源列表。可以点击每个资源以查看请求和响应的详细分类。如果的浏览器发送了 Cookie，将在请求的“Cookie”选项卡上看到这些 Cookie。
