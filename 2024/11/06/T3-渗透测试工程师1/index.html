什么是渗透测试？

渗透测试的目标是在客户的应用程序或系统中发现潜在的漏洞，以帮助客户提升防御能力。

渗透测试（pentest）是一种遵循道德准则的安全评估方式，其核心是测试和分析防御机制的有效性，目标在于提升整体的安全性。

在渗透测试过程中，测试人员会使用与恶意攻击者类似的工具、技术和方法，以一种类似审计的方式来评估系统的弱点。

渗透测试的伦理规范

渗透测试是一项经过系统所有者授权的安全评估工作，其目标是审查系统的安全防御机制。渗透测试的合法性较为明确，任何超出协议范围的行为都被视为未经授权的行为，可能构成违法。

在渗透测试正式开始之前，渗透测试员和系统所有者会进行讨论，明确测试工具、技术及测试范围。这一讨论形成了渗透测试协议的基础，定义了测试的具体流程和边界。

提供渗透测试服务的公司必须遵守相关法律框架和行业认证要求

黑客通常被分为三种类别，其行为动机和道德观决定了他们的类别：

类别 描述
白帽 遵守法律，并利用技能为他人谋福利的黑客。
灰帽 这些黑客有时为公益而行动，但不总是遵守法律或道德标准。
黑帽 犯罪分子，通常以损害他人为代价谋取私利。

交战规则（ROE）

ROE 文件在渗透测试初期制定，包含了三个主要部分，决定了测试的具体执行方式：

部分 描述
许可 明确授权进行测试活动，确保测试人员和公司的法律保护。
测试范围 标明测试适用的目标系统或应用程序。
规则 规定允许的技术手段。

渗透测试的方法论

渗透测试的目标和范围多种多样，因此每次测试都可能有所不同，无法用一种通用方案来规范所有测试步骤。渗透测试员在测试中的步骤被称为方法论，一个有效的方法论应具备灵活性，适应不同的情境。

渗透测试中的通用阶段：

阶段 描述
信息收集 通过公开来源情报（OSINT）和研究，收集尽可能多的公开信息。此阶段不涉及系统扫描。
枚举/扫描 识别系统上运行的应用程序和服务。
漏洞利用 利用系统或应用程序中发现的漏洞，可能涉及公开漏洞利用工具或应用逻辑漏洞。
权限提升 成功利用系统或应用程序后，尝试扩大权限，可横向访问同权限组的其他账户或纵向获得更高权限。
后渗透阶段 包括以下阶段：横向移动寻找其他可攻击的主机、在更高权限下收集更多信息、隐匿行踪以及撰写报告。

OSSTMM

《开放源代码安全测试方法手册》（OSSTMM）提供了一个系统、软件、应用程序、通信和网络安全的人因测试策略框架。该方法论主要关注系统与应用程序间的通信：

电信
有线网络
无线通信

优势 劣势
提供广泛覆盖的测试策略。 框架复杂，细节繁多且术语独特。
包含特定目标的方法论。 无法快速上手，学习成本较高。
可根据组织需求灵活调整。 对新手来说不易理解。
为系统和应用设定了通用的安全标准。

OWASP

开放 Web 应用安全项目（OWASP）是一个由社区推动并定期更新的框架，专门用于 Web 应用程序和服务的安全性测试。OWASP 会定期发布报告，列出 Web 应用程序的十大安全漏洞、测试方法和补救措施。

优势 劣势
易于理解，适合入门。 未必能够清晰分类所有 Web 应用的漏洞类型。
维护活跃，更新频繁。 未针对特定的软件开发生命周期提出建议。
覆盖测试、报告和补救的各个阶段。 缺乏认证。
专注于 Web 应用程序和服务。

NIST 网络安全框架1.1

NIST 网络安全框架用于帮助组织提高网络安全标准并管理网络威胁风险。该框架提供了安全控制和成功基准指导，适用于关键基础设施和商业组织。虽然该框架未直接为渗透测试提供标准化的操作指南，但它包含部分指导原则。

优势 劣势
广泛使用 多个版本，难以确定最适合的版本。
有助于设定标准，减轻网络威胁。 审计政策较弱，难以确定违规事件。
更新频繁。 对云计算考虑不足。
提供认证支持。
可与其他框架协同实施。

NCSC 网络评估框架（CAF）

网络评估框架（CAF）由 NCSC 发布，包含十四项原则，用于评估和管理网络威胁的风险。该框架适用于执行关键服务和活动的组织，重点关注数据安全、系统安全、身份和访问控制、弹性、监控、响应和恢复计划等方面。

优势 劣势
由政府机构支持，权威性高。 较新，许多组织尚未完全适应。
提供认证支持。 基于原则和理念，缺乏操作性指导。
涵盖十四项原则，从安全到应急响应。

黑盒、白盒和灰盒渗透测试

在测试应用程序或服务时，主要有三种测试范围。测试员对目标的了解程度决定了渗透测试的深度。

黑盒测试

黑盒测试测试员在此过程中没有应用程序或服务的内部运作信息。测试员以普通用户的身份测试应用程序的功能和交互性，常见的操作包括与界面交互，以测试预期结果的实现。由于缺乏内部信息，黑盒测试增加了信息收集和枚举阶段的时间，因为测试员需自行识别目标的攻击面。

灰盒测试

灰盒测试结合了黑盒和白盒测试的特点。测试员对应用程序或软件的内部组件有一些有限了解，但仍主要通过黑盒测试的方式与应用程序交互，利用已有知识来解决遇到的问题。这种有限的内部信息可以节省测试时间，尤其适用于已高度强化的攻击面。

白盒测试

白盒测试通常由具备编程和应用逻辑知识的软件开发人员执行。测试员将深入测试应用程序的内部组件，以确保特定功能正常工作并且响应时间合理。白盒测试提供了对应用程序及其预期行为的全面了解，虽然更耗时，但这种全面的知识保障了对整个攻击面的完整验证。
